# 第 3 章 字符串、向量和数组

在前一章内置类型的基础上，本章介绍 C++ 标准库中一组更高级的抽象数据类型：可变长字符串 `string`、可变长集合 `vector`、配套迭代器，以及底层内置数组。

## 3.1 命名空间的 using 声明

- 通过 `using` 声明简化代码（如 `using std::cin;`）
- 使用标准库名字时无需每次都加 `std::` 前缀

## 3.2 标准库类型 string

- 可变长字符序列 `string` 类
- 对象的定义与初始化
- 读写操作、拼接、比较等基本操作
- `cctype` 头文件：`isalpha`、`toupper` 等，单独处理字符串中的特定字符

## 3.3 标准库类型 vector

- C++ 最常用的容器类型，存放给定类型对象的可变长序列
- 定义、列表初始化
- `push_back`：运行时动态添加元素
- 下标运算符的正确用法（**不能用下标添加新元素**）

## 3.4 迭代器介绍

- 迭代器：更通用的访问 `string` 字符和 `vector` 元素的方法
- `begin` / `end`、`cbegin` / `cend` 获取迭代器
- 解引用、递增遍历容器
- 迭代器的算术运算

## 3.5 数组

- C++ 内置数组：大小固定，灵活性不如 `vector`
- 定义、初始化、下标访问
- **指针与数组的紧密联系**：数组名常自动转换为指向首元素的指针
- 现代 C++ 应优先使用 `vector`、`string`，而非低级内置数组和指针

## 3.6 多维数组

- C++ 没有真正的多维数组，所谓多维数组实为「数组的数组」
- 使用范围 `for`、指针、类型别名（`typedef` / `using`）简化复杂多维数组的遍历与操作

## 小结

本章是 C++ 标准库学习的起点。应**优先选用**标准库的 `string` 和 `vector`，只在必要时才考虑底层数组或指针。
