# 第 18 章 用于大型程序的工具

面向**中大型项目、多文件工程、复杂依赖**的 C++ 进阶机制，写小 demo 基本用不到。

## 18.1 异常处理（深入版）

第 5 章仅为入门，本章深入：

- 完整机制：`throw` 抛出、多层函数栈自动回溯、`try-catch` 捕获
- **noexcept**：标记函数是否会抛出异常
- 异常对象、继承体系（`exception`、`runtime_error`、`logic_error` 等）
- 工程准则：RAII 配合异常保证资源不泄漏
- 异常安全三级：**基础保证**、**强保证**、**无异常保证**

## 18.2 命名空间 namespace

解决大型项目**命名冲突**：

- 自定义命名空间、嵌套命名空间、内联命名空间
- `using 声明` vs `using namespace 指令`，滥用全局 using 的坑
- 标准库 `std` 原理、库版本兼容、第三方库整合避重名

## 18.3 多重继承与虚继承

1. **多重继承**：一个类继承多个父类 → 菱形继承、成员名冲突、二义性
2. **菱形继承**：公共祖先多份拷贝、数据冗余
3. **虚继承 `virtual public`**：公共基类只保留一份实例
4. 多重继承下的构造/析构顺序、类型转换、虚函数重写规则

## 18.4 运行时类型识别 RTTI

| 运算符 | 用途 |
|--------|------|
| `dynamic_cast` | 安全向下转型，失败返回 `nullptr` |
| `typeid` | 获取 `type_info` 类型信息 |

适用：多态体系里判断实际对象类型。开销略大，能靠虚函数解决就尽量不用 RTTI。

## 学习优先级

**中大型项目架构必学**：异常、命名空间、多重继承、RTTI 是工程基础。
