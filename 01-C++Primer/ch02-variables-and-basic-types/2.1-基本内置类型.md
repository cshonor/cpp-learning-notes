# 2.1 基本内置类型

## 本节讲什么

介绍 C++ 定义的**算术类型**（包括字符、整型数、布尔值和浮点数）以及**空类型**（`void`）。详细讲解**字面值常量**（literal）：十进制、八进制、十六进制的整型字面值，以及字符和字符串字面值等。

## 要点

- **算术类型**：`char`、`int`、`long`、`bool`、`float`、`double` 等，不同机器上大小可能不同
- **void**：不对应具体对象，常用于函数无返回值或通用指针
- **整型字面值**：十进制 `42`；八进制 `042`（前缀 `0`）；十六进制 `0x2A`（前缀 `0x`/`0X`）
- **浮点字面值**：**不带后缀默认是 `double`**（如 `3.14` → 8 字节）；`3.14f` / `3.14F` → `float`；`3.14L` / `3.14l` → `long double`
- **字符字面值**：`'a'`；**字符串字面值**：`"hello"`（实际含末尾 `\0`）
- **转义序列**：`\n`、`\t`、`\\`、`\"` 等
- **类型大小因平台而异**：C++ 标准只规定各类型的**最小取值范围**，**没有强制字节数**；不同硬件架构、编译器下同一类型可能占不同字节

## 为什么同一类型在不同机器上大小不一样？

根本原因是**硬件架构**和**编译器/ABI 规则**不同。标准保证「至少能表示多大范围」，具体占几字节由实现决定。

| 类型 | 常见情况 | 注意 |
|------|----------|------|
| **`int`** | 32 位系统通常 4 字节 | 16 位旧系统可能是 2 字节 |
| **`long`** | Windows 32/64 位通常都是 **4 字节** | Linux 64 位上通常是 **8 字节** |
| **`long long`** | 一般 8 字节 | 跨平台相对更统一 |
| **`bool`** | 通常 1 字节 | 个别嵌入式平台可能自定义 |
| **`float` / `double`** | 通常 4 / 8 字节 | 相对固定，嵌入式也可能有差异 |
| **`char`** | 通常 1 字节 | — |

**跨平台开发时**：不要假设 `int`、`long` 的字节数；需要固定宽度时用 `<cstdint>` 里的 `int32_t`、`int64_t` 等（后续章节会用到）。

## 与 Rust 对照：谁锁死了类型大小？

| | Rust | C++ |
|---|------|-----|
| **原生整数** | `i32` 永远 4 字节，`i64` 永远 8 字节 | `int`/`long` 大小因平台而异 |
| **规则位置** | **语言层面**写死 | **编译器/平台**决定，标准只规定最小范围 |
| **HFT 风险** | 换平台少一层「类型大小变了」的坑 | 需主动用 `int64_t`、`static_assert` 等约束 |
| **C++ 优势** | — | 兼容底层硬件、利用平台原生特性做极致优化时更灵活 |

Rust 从语法就把大小锁死，HFT 里天然少一层风险；C++ 也能用定宽类型 + 编译选项达到同样效果，但**要开发者主动做**。两种都是合理取舍。

## 浮点字面值：默认是 double 还是 float？

**不带后缀的浮点字面量默认就是 `double`。**

| 写法 | 类型 | 常见大小 |
|------|------|----------|
| `3.14` | **`double`** | 通常 8 字节 |
| `3.14f` 或 `3.14F` | **`float`** | 通常 4 字节 |
| `3.14L` 或 `3.14l` | **`long double`** | 平台相关 |

```cpp
float  price = 3.14;    // ⚠️ 3.14 是 double，赋给 float 可能精度损失，编译器常警告
float  price_ok = 3.14f;  // ✅ 明确 float 字面量
double fee = 5.5;       // 5.5 默认就是 double
```

HFT 里若变量是 `float`，字面量记得加 **`f` 后缀**，避免隐式从 `double` 窄化到 `float` 的警告或意外精度问题。

## 字符串与 `'\0'`（空字符）

**双引号字符串 `"..."` 里，末尾的 `'\0'` 是编译器自动加的，一般不用手写。**

```cpp
"abc"   // 内存里实际是：'a' 'b' 'c' '\0'  共 4 字节
```

| 写法 | 含义 |
|------|------|
| `"abc"` | 编译器自动在末尾加 `'\0'`，表示字符串结束 |
| `"abc\0def"` | **第一个** `'\0'` 就当作结束符；后面 `def` 仍在内存里，但 `cout` 打印只显示 `abc` |
| `'\0'`（单引号） | **单个字符字面量**，ASCII 码为 0 的空字符；和字符串末尾结束符**值相同**，用法不同 |

```cpp
#include <iostream>
int main() {
    const char *s1 = "abc";
    const char *s2 = "abc\0def";

    std::cout << s1 << std::endl;  // abc
    std::cout << s2 << std::endl;  // abc（遇到第一个 \0 就停）
    std::cout << s2[4];            // d（内存里还有，但 cout 按 C 字符串规则遇 \0 截断）

    char c = '\0';                 // 空字符本身，不是字符串
    return 0;
}
```

**记法**：双引号是「字符串」，编译器帮你收尾 `'\0'`；单引号是「一个字符」，`'\0'` 就是那个值为 0 的字符。HFT / 二进制协议里若自己组包，别假设 `std::string` 和 C 字符串规则完全一样——需要定长或显式长度时要用别的方式（后面 `string` 章节会讲）。

## 转义字符是什么？

**转义字符** = 反斜杠 `\` + 一个普通字符，用来表示**看不见**或**和语法冲突**的字符。源码里没法直接敲换行写进字符串，就用 `\n` 代替。

| 写法 | 含义 | 干嘛用 |
|------|------|--------|
| `\n` | 换行 | `cout << "a\nb"` 打印两行 |
| `\t` | 制表符（Tab） | 对齐文本 |
| `\\` | 一个真正的 `\` | 路径 `"C:\\test"`，否则 `\t` 会被当成 Tab |
| `\"` | 双引号本身 | 字符串里包含 `"` |
| `\'` | 单引号本身 | 字符/字符串里包含 `'` |
| `\0` | 空字符（ASCII 0） | 见上一节 |

```cpp
std::cout << "第一行\n第二行\t对齐" << std::endl;
std::cout << "路径: C:\\Users\\test" << std::endl;
```

**和 `\0` 的关系**：`\n`、`\t` 也是转义序列，只是它们表示的是**有具体作用的控制字符**；`\0` 表示字符串结束。都在双引号字符串里用。

**HFT 注意**：转义规则是**源码/文本**里的写法；二进制协议传原始字节时，别把数据里的 `0x0A` 和源码里的 `\n` 混为一谈——那是两回事。

## 开发阶段怎么「掐死」类型不符合预期？

### 1. `sizeof` + `static_assert`（编译期检查）

`sizeof(T)` 在编译期可知（对大多数类型）。配合 **`static_assert`**（C++11），可以在**编译阶段**就报错，而不是等到运行时才发现：

```cpp
#include <cstdint>

// 本项目假定 long 为 8 字节；不符合则编译直接失败
static_assert(sizeof(long) == 8, "本项目要求 long 为 8 字节");

// 定宽类型更可预期
static_assert(sizeof(int64_t) == 8, "int64_t 必须为 8 字节");
```

HFT / 嵌入式项目里常在头文件或公共配置里写这类断言，避免换平台后静默出错。

### 2. `std::numeric_limits`（查类型能表示的范围）

`<limits>` 里的 **`std::numeric_limits<T>`** 可获取当前环境下类型的极值，例如：

```cpp
#include <limits>
#include <iostream>

int main() {
    std::cout << "long max: " << std::numeric_limits<long>::max() << std::endl;
    std::cout << "long min: " << std::numeric_limits<long>::min() << std::endl;

    constexpr long kMaxRpm = 50000;  // 业务：电机转速上限
    static_assert(kMaxRpm <= std::numeric_limits<long>::max(),
                  "long 装不下所需转速范围，请改用 int64_t");
    return 0;
}
```

**业务用法（无人机电机转速）**：先查 `numeric_limits` 看当前类型能否承载所需范围；不够就换成 **`int64_t`** 等定宽类型，再配合 `static_assert(sizeof(int64_t) == 8)` 锁定布局。

| 手段 | 时机 | 作用 |
|------|------|------|
| `sizeof` + `static_assert` | **编译期** | 字节数不对 → 直接编译失败 |
| `numeric_limits<T>::max/min` | 编译期或**运行时** | 判断取值范围是否够用 |
| `int32_t` / `int64_t` | 设计时 | 跨平台固定宽度，协议/HFT 常用 |

## 示例

```cpp
#include <iostream>
#include <limits>
int main() {
    int dec = 42;
    int oct = 052;       // 八进制，值为 42
    int hex = 0x2A;      // 十六进制，值为 42

    double pi = 3.14;        // 字面量 3.14 默认是 double
    float  ratio = 0.5f;     // 明确 float 用 f 后缀

    bool flag = true;
    char ch = 'A';
    const char *msg = "hello";

    std::cout << dec << " " << oct << " " << hex << std::endl;
    std::cout << pi << " " << flag << " " << ch << " " << msg << std::endl;

    // 查看本机各类型占多少字节（结果因平台而异）
    std::cout << "sizeof(int)=" << sizeof(int)
              << " sizeof(long)=" << sizeof(long)
              << " sizeof(double)=" << sizeof(double) << std::endl;

    std::cout << "long max=" << std::numeric_limits<long>::max() << std::endl;

    return 0;
}
```
