# 2.2 变量

## 本节讲什么

讲解变量的**定义与初始化**（特别强调初始化与赋值是两个不同的操作）。为支持分离式编译，深入区分变量的**声明（declaration）**和**定义（definition）**，介绍 `extern` 关键字的作用。同时阐述标识符的命名规范以及作用域（全局作用域、局部作用域）的嵌套规则。

## 要点

- **初始化 vs 赋值**：定义时直接给初值是初始化；已有对象再赋新值是赋值
- **声明**：说明名字与类型的关系，可多次；**定义**：分配存储，通常只能一次
- **`extern`**：声明变量或函数在其他文件中定义；`extern int i;` 是声明，`extern int i = 0;` 是定义
- **未初始化变量**：局部内置类型变量值未定义，使用是未定义行为
- **标识符**：字母、数字、下划线，不能以数字开头；区分大小写
- **作用域**：内层名字可遮蔽外层同名变量

## 变量 vs 常量

C++ 常按**能不能改**来区分：

| | **变量** | **常量** |
|---|----------|----------|
| **能否修改** | 可以重新赋值 | 定义后不能改（或编译期就定死） |
| **例子** | `int price = 100; price = 200;` | `const int max = 10000;` |
| **关键字** | （无特殊关键字） | `const`、`constexpr` |
| **字面值** | — | `100`、`"hello"`、`3.14` 等直接写在代码里的常量 |

```cpp
int price = 100;              // 变量：可以改
price = 200;

const int max_orders = 10000;   // 常量：运行期不可改
// max_orders = 20000;          // 错误

constexpr int kMaxLatency = 50;   // 编译期常量，HFT 里常用作阈值、上限
```

- **`const`**：值不可修改；常用于参数、引用绑定常量对象
- **`constexpr`**：编译期就能确定的常量，编译器可做优化
- **字面值常量**：如 `42`、`"abc"`，嵌在表达式里，不是带名字的变量

## 怎么查类型的最大值？

不用自己算，用 `<limits>` 里的 **`std::numeric_limits`**（详见 [2.1](./2.1-基本内置类型.md)）：

```cpp
#include <limits>
#include <iostream>

int main() {
    std::cout << "int max:  " << std::numeric_limits<int>::max() << std::endl;
    std::cout << "long max: " << std::numeric_limits<long>::max() << std::endl;
    return 0;
}
```

**注意**：`max()` 是**当前机器、当前编译环境**下该类型的最大值；换平台 `int`/`long` 大小变了，最大值也会变。

## `static` 是什么？（多面手）

`static` 含义**取决于写在哪**：

### 1. 函数内的 `static` 变量

- **只初始化一次**，生命周期延续到**整个程序结束**
- 下次进函数时保留上次的值

```cpp
void submit_order() {
    static int order_count = 0;  // 只初始化一次
    ++order_count;               // 累计总下单次数，不会每次调用都归零
}
```

### 2. 全局 / 命名空间里的 `static`

- 限制作用域为**当前源文件**（内部链接）
- 其他 `.cpp` 看不到这个名字，避免多文件同名冲突
- HFT 多文件项目里常用来**隐藏**本文件内部实现

```cpp
// foo.cpp
static int internal_counter = 0;  // 仅 foo.cpp 可见
static void helper() { /* ... */ }
```

（现代 C++ 也常用**匿名命名空间**达到类似效果。）

### 3. 类里的 `static` 成员（预告）

- 属于**类本身**，不属于某个对象；所有对象**共享一份**
- 例：策略类里 `static` 的延迟阈值，所有实例共用
- 详细见第 7 章「类的静态成员」

```cpp
class Strategy {
public:
    static int max_latency_ms;  // 类级别，共享
};
```

| 位置 | 作用 |
|------|------|
| 函数内 | 只初始化一次，程序全程存活 |
| 文件作用域 | 仅本文件可见，隐藏实现 |
| 类内 | 所有对象共享的类级数据 |

## `extern` 关键字完整解析

支持**分离式编译**：多个 `.cpp` 分别编译，最后链接成一个程序。`extern` 用来声明「这个名字的定义在别处」。

### 通俗理解：external（外部的）

`extern` = **声明，不是定义**。告诉编译器：

> 这个函数 / 全局变量的**本体不在当前 `.cpp`**，在别的源文件里；编译本文件时先「放行」，**链接阶段**再去别的 `.cpp` 里把它找出来拼上。

| | 定义 | `extern` 声明 |
|---|------|----------------|
| 内存 | ✅ 分配 | ❌ 不分配 |
| 次数 | 全项目**一次** | 可多次 |
| 例子 | `int g_num = 10;` | `extern int g_num;` |

**文件 A.cpp / B.cpp 例子：**

```cpp
// A.cpp
int g_num = 100;   // 定义：内存在这里

// B.cpp
extern int g_num;  // 声明：g_num 在别的文件
#include <iostream>
int main() {
    std::cout << g_num;  // 编译时只知道「有个 int g_num」
    return 0;            // 链接时找到 A.cpp 里的实体，拼在一起
}
```

**函数可以省略 `extern`**：C++ 里函数声明**默认隐式带 `extern`**，所以跨文件调函数一般只写 `void func();` 即可。

**全局变量跨文件必须用 `extern` 声明**（或在头文件里 `extern` 声明后 `#include`），否则编译器不知道去别处找。

### 1. 声明 vs 定义

| | **声明（declaration）** | **定义（definition）** |
|---|-------------------------|------------------------|
| **做什么** | 告诉编译器类型和名字 | 分配内存（变量）或提供函数体 |
| **出现次数** | 可多次 | 整个程序**只能一处** |
| **不写的后果** | — | 多处定义 → 链接报 `multiple definition` |

### 2. `extern` 修饰变量：两种写法

**`extern int i;` —— 纯声明（引用性声明）**

- **不分配内存**
- 表示：`i` 的定义在**其他 `.cpp`** 或本文件更后面
- 可在头文件、多个源文件中重复写，无冲突

```cpp
// a.cpp
int i = 100;        // 唯一定义，分配内存

// b.cpp
extern int i;       // 仅声明，借用 a.cpp 的 i
#include <iostream>
void f() { std::cout << i; }
```

**`extern int i = 0;` —— 带初始化 = 定义（抵消 extern）**

C/C++ 规则：**带显式初始化的 `extern` 一律视为定义**，等价于 `int i = 0;`：

- 会分配全局内存
- 整个项目只能写**一次**
- 违背「引用外部变量」的本意，工程中**极少这么写**

### 3. 易混对比

```cpp
int i;               // 定义（C++ 中尽量避免；全局默认零初始化）
extern int i;        // 仅声明，无内存
int i = 0;           // 定义
extern int i = 0;    // 定义，等价上一行
```

**一句话**：`extern int i;` 只有声明含义；只要 `extern` 变量附带 `= 初值`，就是定义。

### 4. `extern` 修饰函数

函数更简单——**声明默认就是 extern，可省略**：

```cpp
extern void func();  // 声明，无函数体
void func();         // 完全等价
```

**有 `{}` 函数体 = 定义**，与是否写 `extern` 无关：

```cpp
void func() { }              // 定义
extern void func() { }       // 同样是定义，写 extern 无意义
```

### 5. 工程规范（头文件 + 单源文件定义）

**头文件只写 extern 声明：**

```cpp
// global.h
extern int g_val;    // 仅声明，多文件 #include 无冲突
```

**只有一个 `.cpp` 写定义：**

```cpp
// global.cpp
int g_val = 10;      // 唯一定义，分配内存
```

**禁止在头文件写：**

```cpp
extern int g_val = 10;   // ❌ 定义！头文件被多个 .cpp 包含 → 重复定义
int g_val = 10;          // ❌ 同上
```

HFT 多文件项目里：全局配置、共享计数器等用「`.h` 声明 + 一个 `.cpp` 定义」是标准做法。

### 6. 什么时候会「重定义」？怎么规避？

| 错误写法 | 原因 |
|----------|------|
| 两个 `.cpp` 都写 `int g_num = 10;` | 两处定义 → 链接 `multiple definition` |
| 头文件写 `int g_num = 10;` 并被多个 `.cpp` 包含 | 每个 `.cpp` 都生成一份定义 |
| 头文件写 `extern int g_num = 10;` | 带 `=` 仍是定义，同上 |

**规避**：`.h` 里只写 `extern int g_num;`，**恰好一个** `.cpp` 写 `int g_num = 10;`。

### 7. 补充：`extern "C"`（与变量 `extern` 不同）

`extern "C"` 用来**关闭 C++ 名字修饰（name mangling）**，让 C++ 能调用 C 写的函数。DPDK、Linux 内核接口里很常见：

```cpp
extern "C" {
    void dpdk_init();   // 按 C 规则链接，符号名不被 C++ 改写
}
```

这是**链接方式**的声明，和「全局变量 `extern int g_num`」不是同一类语法，后面第 18 章 / 与 C 混编时会再遇到。

## 示例

```cpp
#include <iostream>
#include <limits>

int global_val = 10;

extern int shared_val;  // 声明：在其他 .cpp 中定义

void demo_static() {
    static int call_count = 0;
    ++call_count;
    std::cout << "called " << call_count << " times\n";
}

int main() {
    int ival = 0;
    const int limit = 100;
    constexpr int kMax = 10000;

    demo_static();
    demo_static();  // call_count 累加，不会重置

    std::cout << "int max=" << std::numeric_limits<int>::max() << std::endl;
    std::cout << ival << " " << limit << " " << kMax << std::endl;
    return 0;
}
```
