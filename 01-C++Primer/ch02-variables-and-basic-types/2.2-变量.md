# 2.2 变量

## 本节讲什么

讲解变量的**定义与初始化**（特别强调初始化与赋值是两个不同的操作）。为支持分离式编译，深入区分变量的**声明（declaration）**和**定义（definition）**，介绍 `extern` 关键字的作用。同时阐述标识符的命名规范以及作用域（全局作用域、局部作用域）的嵌套规则。

## 要点

- **初始化 vs 赋值**：定义时直接给初值是初始化；已有对象再赋新值是赋值
- **声明**：说明名字与类型的关系，可多次；**定义**：分配存储，通常只能一次
- **`extern`**：声明变量或函数在其他文件中定义；`extern int i;` 是声明，`extern int i = 0;` 是定义
- **未初始化变量**：局部内置类型变量值未定义，使用是未定义行为
- **作用域**：内层名字可遮蔽外层同名变量（详见 [变量遮蔽](#变量遮蔽variable-shadowing)）

## 标识符命名规则

**标识符** = 变量、函数、类型等的名字。编译器**强制校验**，违反直接报错。

### 硬性规则（标准规定）

1. **允许字符**：仅 **a–z、A–Z、0–9、下划线 `_`**
2. **首字符**：只能是**字母或 `_`**，**不能以数字开头**
3. **大小写敏感**：`num`、`Num`、`NUM` 是三个不同名字
4. **附加约束**：
   - 不能与**关键字**重名（`int`、`if`、`for`、`return` 等）
   - 内部不能有空格、Tab、换行
   - **`__` 开头、`_` + 大写字母** 等属于保留名，业务代码避免使用

### 合法 / 非法示例

```cpp
// ✅ 合法
age  Age  _age  age123  student_score  MAX_NUM

// ❌ 非法
// 123age         数字开头
// student-score  含减号
// student score  含空格
// int            关键字
// $num           非法字符
```

### 大小写独立

```cpp
int a = 10;
int A = 20;
std::cout << a << " " << A;   // 10 20，完全独立
```

### 工程命名习惯（非语法强制，提升可读性）

| 对象 | 常见风格 | 例子 |
|------|----------|------|
| 局部变量、普通函数 | 小写 + 下划线 | `user_id`、`submit_order` |
| 宏 / 宏常量 | 全大写 | `MAX_LEN` |
| C++ 类 / 结构体 | 大驼峰 | `StudentInfo`、`SalesItem` |
| 循环变量 | 单字母可接受 | `i`、`j`、`k` |

**一句话**：只能由**字母、数字、下划线**组成，**首字符不能是数字**，**区分大小写**，不能用关键字。

## 变量 vs 常量

C++ 常按**能不能改**来区分：

| | **变量** | **常量** |
|---|----------|----------|
| **能否修改** | 可以重新赋值 | 定义后不能改（或编译期就定死） |
| **例子** | `int price = 100; price = 200;` | `const int max = 10000;` |
| **关键字** | （无特殊关键字） | `const`、`constexpr` |
| **字面值** | — | `100`、`"hello"`、`3.14` 等直接写在代码里的常量 |

```cpp
int price = 100;              // 变量：可以改
price = 200;

const int max_orders = 10000;   // 常量：运行期不可改
// max_orders = 20000;          // 错误

constexpr int kMaxLatency = 50;   // 编译期常量，HFT 里常用作阈值、上限
```

- **`const`**：值不可修改；常用于参数、引用绑定常量对象
- **`constexpr`**：编译期就能确定的常量，编译器可做优化
- **字面值常量**：如 `42`、`"abc"`，嵌在表达式里，不是带名字的变量

## 怎么查类型的最大值？

不用自己算，用 `<limits>` 里的 **`std::numeric_limits`**（详见 [2.1](./2.1-基本内置类型.md)）：

```cpp
#include <limits>
#include <iostream>

int main() {
    std::cout << "int max:  " << std::numeric_limits<int>::max() << std::endl;
    std::cout << "long max: " << std::numeric_limits<long>::max() << std::endl;
    return 0;
}
```

**注意**：`max()` 是**当前机器、当前编译环境**下该类型的最大值；换平台 `int`/`long` 大小变了，最大值也会变。

## `static` 是什么？（多面手）

`static` 含义**取决于写在哪**：

### 1. 函数内的 `static` 变量

- **只初始化一次**，生命周期延续到**整个程序结束**
- 下次进函数时保留上次的值

```cpp
void submit_order() {
    static int order_count = 0;  // 只初始化一次
    ++order_count;               // 累计总下单次数，不会每次调用都归零
}
```

### 2. 全局 / 命名空间里的 `static`

- 限制作用域为**当前源文件**（内部链接）
- 其他 `.cpp` 看不到这个名字，避免多文件同名冲突
- HFT 多文件项目里常用来**隐藏**本文件内部实现

```cpp
// foo.cpp
static int internal_counter = 0;  // 仅 foo.cpp 可见
static void helper() { /* ... */ }
```

（现代 C++ 也常用**匿名命名空间**达到类似效果。）

### 3. 类里的 `static` 成员（预告）

- 属于**类本身**，不属于某个对象；所有对象**共享一份**
- 例：策略类里 `static` 的延迟阈值，所有实例共用
- 详细见第 7 章「类的静态成员」

```cpp
class Strategy {
public:
    static int max_latency_ms;  // 类级别，共享
};
```

| 位置 | 作用 |
|------|------|
| 函数内 | 只初始化一次，程序全程存活 |
| 文件作用域 | 仅本文件可见，隐藏实现 |
| 类内 | 所有对象共享的类级数据 |

## 变量遮蔽（Variable Shadowing）

**变量遮蔽**：在**嵌套作用域**（代码块、函数、循环）里，内层重新声明与外层**同名**的变量，内层名字会**屏蔽**外层同名变量；在内层范围内，所有对那个名字的引用只指向内层变量，外层同名变量暂时不能直接访问。

**词法作用域查找链（由内向外）：**

1. 先在**当前最内层**找名字，找到就用
2. 找不到再向外层回溯
3. 内层一旦定义同名变量，查找在此终止，**不会**再匹配外层

| 情况 | 结果 |
|------|------|
| **同一 `{}` 内两次 `int a;`** | ❌ 语法错误（重定义） |
| **嵌套 `{}` 内外各一个 `int a;`** | ✅ 合法，内层**遮蔽**外层，两块独立内存 |

### 块级嵌套示例

```cpp
#include <iostream>
int main() {
    int i = 0;
    std::cout << "外层 i = " << i << std::endl;  // 0

    {
        int i = 7;   // 内层 i 遮蔽外层 i
        int j = 9;
        std::cout << "内层 i = " << i << ", j = " << j << std::endl;  // 7, 9
    }   // 内层 i 销毁，遮蔽结束

    std::cout << "外层 i 恢复 = " << i << std::endl;  // 0
    return 0;
}
```

### 四种常见场景

**1. 局部变量遮蔽全局变量**

```cpp
int x = 100;

void func() {
    int x = 50;              // 局部遮蔽全局
    std::cout << x;          // 50
}

int main() {
    func();
    std::cout << x;          // 100
}
```

**2. 内层块遮蔽形参**（同一函数体直接 `int a` 与形参同名会**重定义报错**，须包一层 `{}`）

```cpp
void test(int a) {
    std::cout << a;          // 5（形参）
    {
        int a = 20;        // 内层块遮蔽形参
        std::cout << a;    // 20
    }
    std::cout << a;          // 5（形参恢复可见）
}
```

**3. 循环体内遮蔽循环变量**（C++11 起 `for (int i ...)` 的 `i` 仅在循环作用域内）

```cpp
for (int i = 1; i < 3; ++i) {
    int i = 99;              // 循环体内局部 i 遮蔽循环头 i
    std::cout << i << " ";   // 99 99
}
```

**4. 多层嵌套连续遮蔽**

```cpp
int a = 1;
{
    int a = 2;
    {
        int a = 3;
        std::cout << a;    // 3
    }
    std::cout << a;        // 2
}
std::cout << a;            // 1
```

### C++ 特有：访问被遮蔽的全局变量

用**全局作用域解析符 `::`** 穿透遮蔽：

```cpp
int x = 10;
int main() {
    int x = 20;
    std::cout << x << std::endl;   // 20（内层）
    std::cout << ::x << std::endl; // 10（全局）
}
```

### 工程上的问题与规范

**问题：**

1. 改内层变量，外层同名变量**完全不变**，容易误以为改到了外层
2. 可读性差，难一眼看出用的是哪一层
3. 内层变量出 `{}` 即销毁，生命周期易混淆

**建议：**

1. 避免嵌套作用域用同名变量（如全局 `g_x`、局部 `local_x`）
2. 不要与形参同名；若必须区分，用不同名字而非遮蔽
3. 循环体内不要重复定义与控制变量同名的变量

**一句话**：查找**由内向外**；内层一旦声明同名标识符就**遮蔽**外层，内层范围内只能访问内层那份。

## `extern` 关键字完整解析

支持**分离式编译**：多个 `.cpp` 分别编译，最后链接成一个程序。`extern` 用来声明「这个名字的定义在别处」。

### 通俗理解：external（外部的）

`extern` = **声明，不是定义**。告诉编译器：

> 这个函数 / 全局变量的**本体不在当前 `.cpp`**，在别的源文件里；编译本文件时先「放行」，**链接阶段**再去别的 `.cpp` 里把它找出来拼上。

| | 定义 | `extern` 声明 |
|---|------|----------------|
| 内存 | ✅ 分配 | ❌ 不分配 |
| 次数 | 全项目**一次** | 可多次 |
| 例子 | `int g_num = 10;` | `extern int g_num;` |

**文件 A.cpp / B.cpp 例子：**

```cpp
// A.cpp
int g_num = 100;   // 定义：内存在这里

// B.cpp
extern int g_num;  // 声明：g_num 在别的文件
#include <iostream>
int main() {
    std::cout << g_num;  // 编译时只知道「有个 int g_num」
    return 0;            // 链接时找到 A.cpp 里的实体，拼在一起
}
```

**函数可以省略 `extern`**：C++ 里函数声明**默认隐式带 `extern`**，所以跨文件调函数一般只写 `void func();` 即可。

**全局变量跨文件必须用 `extern` 声明**（或在头文件里 `extern` 声明后 `#include`），否则编译器不知道去别处找。

### 1. 声明 vs 定义

| | **声明（declaration）** | **定义（definition）** |
|---|-------------------------|------------------------|
| **做什么** | 告诉编译器类型和名字 | 分配内存（变量）或提供函数体 |
| **出现次数** | 可多次 | 整个程序**只能一处** |
| **不写的后果** | — | 多处定义 → 链接报 `multiple definition` |

### 2. `extern` 修饰变量：两种写法

**`extern int i;` —— 纯声明（引用性声明）**

- **不分配内存**
- 表示：`i` 的定义在**其他 `.cpp`** 或本文件更后面
- 可在头文件、多个源文件中重复写，无冲突

```cpp
// a.cpp
int i = 100;        // 唯一定义，分配内存

// b.cpp
extern int i;       // 仅声明，借用 a.cpp 的 i
#include <iostream>
void f() { std::cout << i; }
```

**`extern int i = 0;` —— 带初始化 = 定义（抵消 extern）**

C/C++ 规则：**带显式初始化的 `extern` 一律视为定义**，等价于 `int i = 0;`：

- 会分配全局内存
- 整个项目只能写**一次**
- 违背「引用外部变量」的本意，工程中**极少这么写**

### 3. 易混对比

```cpp
int i;               // 定义（C++ 中尽量避免；全局默认零初始化）
extern int i;        // 仅声明，无内存
int i = 0;           // 定义
extern int i = 0;    // 定义，等价上一行
```

**一句话**：`extern int i;` 只有声明含义；只要 `extern` 变量附带 `= 初值`，就是定义。

### 4. `extern` 修饰函数

函数更简单——**声明默认就是 extern，可省略**：

```cpp
extern void func();  // 声明，无函数体
void func();         // 完全等价
```

**有 `{}` 函数体 = 定义**，与是否写 `extern` 无关：

```cpp
void func() { }              // 定义
extern void func() { }       // 同样是定义，写 extern 无意义
```

### 5. 工程规范（头文件 + 单源文件定义）

**头文件只写 extern 声明：**

```cpp
// global.h
extern int g_val;    // 仅声明，多文件 #include 无冲突
```

**只有一个 `.cpp` 写定义：**

```cpp
// global.cpp
int g_val = 10;      // 唯一定义，分配内存
```

**禁止在头文件写：**

```cpp
extern int g_val = 10;   // ❌ 定义！头文件被多个 .cpp 包含 → 重复定义
int g_val = 10;          // ❌ 同上
```

HFT 多文件项目里：全局配置、共享计数器等用「`.h` 声明 + 一个 `.cpp` 定义」是标准做法。

### 6. 什么时候会「重定义」？怎么规避？

| 错误写法 | 原因 |
|----------|------|
| 两个 `.cpp` 都写 `int g_num = 10;` | 两处定义 → 链接 `multiple definition` |
| 头文件写 `int g_num = 10;` 并被多个 `.cpp` 包含 | 每个 `.cpp` 都生成一份定义 |
| 头文件写 `extern int g_num = 10;` | 带 `=` 仍是定义，同上 |

**规避**：`.h` 里只写 `extern int g_num;`，**恰好一个** `.cpp` 写 `int g_num = 10;`。

### 7. 补充：`extern "C"`（与变量 `extern` 不同）

`extern "C"` 用来**关闭 C++ 名字修饰（name mangling）**，让 C++ 能调用 C 写的函数。DPDK、Linux 内核接口里很常见：

```cpp
extern "C" {
    void dpdk_init();   // 按 C 规则链接，符号名不被 C++ 改写
}
```

这是**链接方式**的声明，和「全局变量 `extern int g_num`」不是同一类语法，后面第 18 章 / 与 C 混编时会再遇到。

## `#include` 与 `extern`：分工与配合

两者**原理完全不同**，工程里却经常**搭档使用**：

| | **`#include`** | **`extern`** |
|---|----------------|--------------|
| **做什么** | **文本复制**：把头文件内容粘贴进当前 `.cpp` | **一条声明**：变量本体在别的 `.cpp`，不复制实现 |
| **头文件里常放** | 函数声明、`extern` 变量声明、结构体、宏 | 全局变量的 `extern` 声明本身 |

**一句话**：`#include` 负责把 `extern` 声明**导入**本文件；`extern` 负责告诉编译器变量**实体在别的 `.cpp`**。

### 标准组合写法（三文件套路）

**`global.h`**（被 `#include`）

```cpp
extern int g_val;   // 声明全局变量
void test();        // 函数声明（默认 extern，可省略 extern）
```

**`global.cpp`**（唯一定义）

```cpp
int g_val = 100;    // 真正分配内存
void test() { }
```

**`main.cpp`**

```cpp
#include "global.h"   // 把声明拿过来，不用手写 extern

int main() {
    g_val = 200;
    test();
    return 0;
}
```

### 直白区别

1. **函数**：声明写进 `.h`，`#include` 引入即可，一般**不用**写 `extern`（函数声明默认 extern）。
2. **全局变量**：头文件里写 **`extern int g_val;`**，再 `#include` 给其他 `.cpp` 共用。
   - 也可以不在头文件，而在 `main.cpp` 手写 `extern int g_val;`，但项目一大就乱，工程上**统一放头文件**。

### 反面例子（必踩坑）

```cpp
// bad.h  ❌ 千万不要这样
int g_val = 100;    // 这是定义！不是声明

// a.cpp
#include "bad.h"

// b.cpp
#include "bad.h"    // 两个 .cpp 各有一份 g_val 定义 → 链接重定义报错
```

**正确**：头文件只写 `extern int g_val;`，**一个** `global.cpp` 写 `int g_val = 100;`。

## 示例

```cpp
#include <iostream>
#include <limits>

int global_val = 10;

extern int shared_val;  // 声明：在其他 .cpp 中定义

void demo_static() {
    static int call_count = 0;
    ++call_count;
    std::cout << "called " << call_count << " times\n";
}

int main() {
    int ival = 0;
    const int limit = 100;
    constexpr int kMax = 10000;

    demo_static();
    demo_static();  // call_count 累加，不会重置

    std::cout << "int max=" << std::numeric_limits<int>::max() << std::endl;
    std::cout << ival << " " << limit << " " << kMax << std::endl;
    return 0;
}
```
