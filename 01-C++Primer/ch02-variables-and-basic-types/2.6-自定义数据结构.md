# 2.6 自定义数据结构

## 本节讲什么

作为类的初步体验，带领读者使用 `struct` 将相关数据元素组织成自定义类型（如 `Sales_data`）。介绍 C++11 **类内初始值**、**.h + .cpp 分离编译**，以及 **头文件保护**（`#ifndef` / `#pragma once`）。

## 要点

- **`struct`**：打包多种类型成员；默认 **public**（`class` 默认 private）
- **类内初始值**（C++11）：成员声明处 `= 0` 或 `{}`；构造初始化列表优先
- **`.h`** 放声明，**`.cpp`** 放实现；头文件勿写会分配存储的定义
- **头文件保护**：跨平台用 `#ifndef`；现代项目可用 `#pragma once`

## 一、`struct` 结构体

1. **作用**：将一组**不同类型**的数据成员打包，形成自定义聚合类型（C/C++ 通用）
2. **访问权限**：`struct` 默认 **public**；`class` 默认 **private**（其余能力几乎相同）
3. **简单示例**：

```cpp
struct Sales_data {
    std::string bookNo;
    unsigned units_sold = 0;   // C++11 类内默认初始值
    double revenue = 0.0;
};
```

4. **聚合类型**：无自定义构造、无私有成员、无继承、无虚函数等条件下，属于**聚合**，支持 `{}` 聚合初始化：

```cpp
Sales_data d{"978-0-201-78345-0", 25, 22.5};
```

## 二、C++11 类内初始值（就地成员初始化）

### 语法

成员声明时直接赋值：

```cpp
struct S {
    int a = 10;
    std::string str{"test"};
};
```

### 优先级

**构造函数初始化列表 > 类内默认值**

```cpp
struct S {
    int x = 100;
    S() : x(1) {}   // 最终 x 为 1
};
```

### 限制

- **引用成员**、**const 成员**、部分 **静态成员** 不能仅靠类内初始值，须在构造函数初始化列表中初始化
- 成员类型不能是 `auto`

### 优势

减少构造函数里重复的默认值逻辑，统一对象默认状态。

## 三、头文件分离式编译（`.h` + `.cpp`）

### 文件分工

| 文件 | 放什么 |
|------|--------|
| **`.h` / `.hpp`** | **声明**：struct/类定义、函数声明、`extern` 变量、模板、`inline` 函数 |
| **`.cpp`** | **定义/实现**：成员函数体、全局变量定义、普通函数实现 |

### 分离示例

**Sales_data.h**

```cpp
#ifndef SALES_DATA_H
#define SALES_DATA_H

#include <string>

struct Sales_data {
    std::string bookNo;
    unsigned units_sold = 0;
    double revenue = 0.0;
    double avg_price() const;   // 仅声明
};

#endif
```

**Sales_data.cpp**

```cpp
#include "Sales_data.h"

double Sales_data::avg_price() const {
    return units_sold ? revenue / units_sold : 0;
}
```

**main.cpp**

```cpp
#include "Sales_data.h"
#include <iostream>

int main() {
    Sales_data data1;
    Sales_data data2{"978-0-201-78345-0", 25, 22.5};
    std::cout << data2.bookNo << ": " << data2.avg_price() << std::endl;
    return 0;
}
```

### 核心目的

- 声明与实现分离，隐藏细节
- 多个 `.cpp` 共用同一头文件
- 单文件修改实现 → 不必重编整个工程（见 [1.5 编译产物](../ch01-getting-started/1.5-类简介.md)）

## 四、头文件重复包含保护

### 方案 1：`#ifndef` / `#define` / `#endif`（标准、跨平台）

```cpp
#ifndef SALES_DATA_H
#define SALES_DATA_H

// 头文件全部内容

#endif // SALES_DATA_H
```

- **原理**：第一次包含时宏未定义，展开内容并 `#define`；再次包含时跳过
- **优点**：所有 C/C++ 编译器兼容（含嵌入式、老编译器）
- **缺点**：宏名冲突会误屏蔽；建议 `项目_模块_文件名_H` 保证唯一

### 方案 2：`#pragma once`（编译器扩展，非标准）

```cpp
#pragma once
#include <string>
struct Sales_data { /* ... */ };
```

- **原理**：编译器按物理文件路径只加载一次
- **优点**：一行搞定，无宏名冲突，略快
- **缺点**：非 C++ 标准；极少数老编译器不支持；特殊链接场景可能失效

### 对比速记

| 特性 | `#ifndef` 守卫 | `#pragma once` |
|------|----------------|----------------|
| 标准 | ✅ ISO C/C++ | 编译器扩展 |
| 兼容性 | 全平台 | GCC/Clang/MSVC 等现代环境 |
| 写法 | 三行 + 唯一宏名 | 一行 |
| 冲突风险 | 宏名重复会 bug | 无宏名冲突 |
| 适用 | 跨平台库、嵌入式 | 现代本地项目 |

**工程建议**：对外库 / 跨平台 → `#ifndef`；本地现代项目 → 常用 `#pragma once`。

## 五、高频易错点

1. 头文件里不要写**普通函数实现**、**全局变量定义** → 多文件 `#include` → `multiple definition`
2. 类内初始值需 **C++11+**
3. `struct` 与 `class` 仅默认访问权限不同，都支持类内初始值与分离编译
4. 全局变量在 `.h` 里只写 `extern` 声明（见 [2.2 extern](./2.2-变量.md)）

## 六、背诵极简版

```
struct：打包数据，默认 public，聚合可 {}
类内初始值 C++11：成员 = 默认值；构造列表优先
.h 声明  .cpp 实现  头文件别写定义
#ifndef 跨平台  #pragma once 现代省事
```
