# 2.6 自定义数据结构

## 本节讲什么

作为类的初步体验，以《C++ Primer》第五版经典示例 **`Sales_data`** 为主线：三成员 `struct`、C++11 **类内初始值**、教材完整版（构造 / 成员函数 / 友元）、**`.h + .cpp` 分离编译**与**头文件保护**。

## 要点

- **`Sales_data`**：`bookNo` + `units_sold` + `revenue`；末尾 **`;`** 不能漏
- **类内初始值**（C++11）：只用 `=` 或 `{}`，**不能用 `()`**；构造列表优先
- **无自定义构造**时：合成默认构造 + 聚合 `{}` 初始化
- **教材完整版**：加构造 / `combine` / `avg_price` / 友元 `add` `read` `print` 后不再是聚合
- **友元前置声明**：`struct` 定义前三行函数原型；类外调用 + 头文件分离必需
- **`.h` 声明、`.cpp` 实现**；头文件保护 `#ifndef` 或 `#pragma once`

## 一、`Sales_data` 入门三成员（课本经典）

标准写法（**`struct` 定义末尾必须有分号**）：

```cpp
#include <string>

struct Sales_data
{
    std::string bookNo;              // 书籍 ISBN 编号
    unsigned units_sold = 0;         // C++11 类内默认：销售册数
    double revenue = 0.0;            // C++11 类内默认：总销售额
};  // ← 分号极易漏，漏了编译报错
```

| 成员 | 类型 | 类内初值 | 默认构造后 |
|------|------|----------|------------|
| `bookNo` | `std::string` | 无 | 空串 `""` |
| `units_sold` | `unsigned` | `= 0` | `0` |
| `revenue` | `double` | `= 0.0` | `0.0` |

无自定义构造函数时，属于**聚合类型**，支持 `{}` 聚合初始化：

```cpp
Sales_data d{"978-0-201-78345-0", 25, 22.5};
```

## 二、C++11 类内初始值（in-class initializer）

### 语法限制

只能用 **`=`** 或 **花括号 `{}`**，**不能用圆括号 `()`**（会被当成成员函数声明）：

```cpp
unsigned units_sold{0};   // ✅
unsigned units_sold = 0;  // ✅
unsigned units_sold(0);   // ❌ 编译器误判为成员函数
```

### 初始化优先级

**构造函数初始值列表 > 类内默认值 > 默认初始化**

```cpp
struct S {
    int x = 100;
    S() : x(1) {}   // 最终 x 为 1
};
```

对入门版 `Sales_data`：

- 构造函数没写某成员 → 用类内 `= 0` / `= 0.0`
- `bookNo` 无类内初值 → 默认初始化为空串

### 无自定义构造函数时

编译器生成**合成默认构造函数**，自动应用类内初始值：

```cpp
Sales_data item;
// item.bookNo     == ""
// item.units_sold == 0
// item.revenue    == 0.0
```

### 其他限制

- **引用**、**const** 成员、部分 **static** 成员须在构造初始化列表中初始化
- 成员类型不能是 `auto`

## 三、《C++ Primer》完整版 `Sales_data`

教材后续版本增加构造函数、成员函数与友元（`add` / `read` / `print`）。**一旦有自定义构造，不再是聚合**，初始化改用构造函数，例如 `Sales_data("978123456", 5, 39.9)`。

```cpp
#include <string>
#include <iostream>

// 友元前置声明
Sales_data add(const Sales_data&, const Sales_data&);
std::ostream& print(std::ostream&, const Sales_data&);
std::istream& read(std::istream&, Sales_data&);

struct Sales_data
{
    Sales_data() = default;
    Sales_data(const std::string& s) : bookNo(s) {}
    Sales_data(const std::string& s, unsigned n, double p)
        : bookNo(s), units_sold(n), revenue(p * n) {}
    Sales_data(std::istream& is);

    std::string isbn() const { return bookNo; }
    Sales_data& combine(const Sales_data& rhs);
    double avg_price() const;

    std::string bookNo;
    unsigned units_sold = 0;
    double revenue = 0.0;

    friend Sales_data add(const Sales_data&, const Sales_data&);
    friend std::ostream& print(std::ostream&, const Sales_data&);
    friend std::istream& read(std::istream&, Sales_data&);
};

Sales_data& Sales_data::combine(const Sales_data& rhs)
{
    units_sold += rhs.units_sold;
    revenue += rhs.revenue;
    return *this;
}

double Sales_data::avg_price() const
{
    if (units_sold)
        return revenue / units_sold;
    return 0;
}

Sales_data::Sales_data(std::istream& is) { read(is, *this); }

std::istream& read(std::istream& is, Sales_data& item)
{
    double price = 0;
    is >> item.bookNo >> item.units_sold >> price;
    item.revenue = price * item.units_sold;
    return is;
}

std::ostream& print(std::ostream& os, const Sales_data& item)
{
    os << item.isbn() << " " << item.units_sold << " "
       << item.revenue << " " << item.avg_price();
    return os;
}

Sales_data add(const Sales_data& lhs, const Sales_data& rhs)
{
    Sales_data sum = lhs;
    sum.combine(rhs);
    return sum;
}
```

**简单使用：**

```cpp
int main()
{
    Sales_data item1;                              // 空 ISBN、0 本、0 收入
    Sales_data item2("978123456", 5, 39.9);        // 5 本，单价 39.9
    print(std::cout, item2);
    return 0;
}
```

## 四、友元前置声明（`Sales_data` 场景）

完整版代码在 `struct Sales_data` **之前**有三行：

```cpp
// 友元前置声明
Sales_data add(const Sales_data&, const Sales_data&);
std::ostream& print(std::ostream&, const Sales_data&);
std::istream& read(std::istream&, Sales_data&);
```

随后在结构体内部用 `friend` 把上述**全局函数**指定为友元，允许它们访问非公有成员（本例成员虽为 public，但 `read` 需写 `item.revenue` 等，教材仍走友元 IO 模式）。

### 什么是友元前置声明

**前置声明**：在 `struct`/`class` 定义完成之前，先在**全局域**写出友元函数的原型。编译顺序：

1. 读到三行函数原型 → 记住签名；`Sales_data` 作为返回/参数类型被**前向声明**为不完整类型
2. 读到 `struct Sales_data { ... friend add; ... }` → `friend` 与前面全局原型绑定，授予访问权限
3. 在 struct 之后写函数定义（函数体）

### 为什么必须有？（`Sales_data` 的两个核心原因）

**1. 函数原型写在 struct 定义之前，签名里已用到 `Sales_data`**

教材把 `add` / `read` / `print` 的声明放在 struct **上面**。没有这三行，就无法在定义 struct 之前写出合法函数原型；后面的

```cpp
Sales_data::Sales_data(std::istream& is) { read(is, *this); }
```

也要求 `read` 在此刻**已经声明**。

**2. 头文件 / 源文件分离：对外暴露 IO 工具接口**

- **`.h`**：三行前置声明 + struct 定义（含 `friend`）
- **`.cpp`**：`add` / `read` / `print` 的函数体

其他 `.cpp`（如 `main.cpp`）只 `#include "Sales_data.h"` 即可调用 `print(cout, item)`，不必关心实现细节。

### 两种友元写法对比

| 写法 | 是否需要类前前置声明 | 类外能否调用 |
|------|---------------------|--------------|
| **全局友元 + 类外定义**（`Sales_data`） | ✅ 需要 | ✅ |
| **类内 `friend` 后直接写函数体** | ❌ | ❌（几乎仅教学用） |

**情况 1：全局友元（教材用法）**

```cpp
Sales_data add(const Sales_data&, const Sales_data&);

struct Sales_data {
    friend Sales_data add(const Sales_data&, const Sales_data&);
};

Sales_data add(const Sales_data& lhs, const Sales_data& rhs) { /* ... */ }
```

函数属于**全局域**，不属于 `Sales_data`；业务代码、`main`、其他文件都要调用 → 必须类前声明。

**情况 2：类内定义友元（不需前置声明，类外不可用）**

```cpp
struct Test {
    friend void show(Test t) {
        std::cout << t.val;
    }
private:
    int val = 10;
};
// show(t);  // 类外调用：未声明
```

### 关键语法规则

1. `friend` 声明**授予**访问权，但**不等于**完整的工程接口布局；教材在 struct 前写全局原型，是为了**类型前向声明 + 分离编译**。
2. C++11 起，非定义的 `friend` 声明会把函数名**注入**外围命名空间；因此「只有 `friend`、没有类前原型」时，struct **之后**的 `main` 里调用 `add(a,b)` **有时能通过**。但 **`Sales_data(std::istream&)` 依赖的 `read` 必须先声明**，且头文件惯例仍要求三行写在 struct 前。
3. `friend` 写在 `public` / `private` **均可**，只影响类成员访问，不影响友元本身。
4. `print` / `read` 是全局 IO 工具，主程序与其他模块频繁调用 → 必须出现在头文件的全局声明区。

### 错误示范：删掉前置声明 + 流构造函数

下面代码删掉 struct 前的三行声明，并保留依赖 `read` 的构造函数实现 —— **必然编译失败**（`read` 未声明）：

```cpp
#include <iostream>
#include <string>

struct Sales_data
{
    Sales_data() = default;
    Sales_data(std::istream& is) { read(is, *this); }  // ❌ read 尚未声明

    friend Sales_data add(const Sales_data&, const Sales_data&);
    friend std::ostream& print(std::ostream&, const Sales_data&);
    friend std::istream& read(std::istream&, Sales_data&);

    std::string bookNo;
    unsigned units_sold = 0;
    double revenue = 0.0;
};

std::istream& read(std::istream& is, Sales_data& item) { /* ... */ return is; }
// 报错：'read' was not declared in this scope（或 equivalent）
```

**正确做法**：在 struct 前补上 `read`（及 `add`、`print`）的函数原型，或把 `Sales_data(std::istream&)` 的**定义**挪到 struct 之后、且放在 `read` 声明之后。

### 简化记忆

```
类外要调用的友元全局函数 → struct 定义前写函数原型 + struct 内 friend
只在类内用的友元 → friend 后直接写函数体，可不要前置声明
Sales_data 的 add/read/print 是全局 IO 工具 → 头文件前三行 + .cpp 实现
```

详见 ch7 [7.3 友元](../ch07-classes/7.3-友元（Friend）.md)。

## 五、`struct` vs `class`

C++ 中二者**唯一语法差异**是默认访问权限：

| | `struct` | `class` |
|---|----------|---------|
| 默认权限 | **public** | **private** |
| 本示例 | 外部可直接 `item.bookNo` | 成员默认私有，外部不可直接访问 |

教材入门用 `struct` 简化读写；后期改为 `class` 封装数据（见 ch7 类）。

## 六、头文件分离式编译（`.h` + `.cpp`）

### 文件分工

| 文件 | 放什么 |
|------|--------|
| **`.h` / `.hpp`** | **声明**：类型定义、函数声明、`extern`、模板、`inline` |
| **`.cpp`** | **定义**：成员函数体、全局变量定义、普通函数实现 |

### 分离示例（完整版骨架）

**Sales_data.h**

```cpp
#ifndef SALES_DATA_H
#define SALES_DATA_H

#include <string>
#include <iostream>

Sales_data add(const Sales_data&, const Sales_data&);
std::ostream& print(std::ostream&, const Sales_data&);
std::istream& read(std::istream&, Sales_data&);

struct Sales_data {
    Sales_data() = default;
    Sales_data(const std::string& s, unsigned n, double p);
    double avg_price() const;
    Sales_data& combine(const Sales_data& rhs);

    std::string bookNo;
    unsigned units_sold = 0;
    double revenue = 0.0;

    friend Sales_data add(const Sales_data&, const Sales_data&);
    friend std::ostream& print(std::ostream&, const Sales_data&);
    friend std::istream& read(std::istream&, Sales_data&);
};

#endif
```

**Sales_data.cpp** — 构造函数与 `avg_price`、`combine`、`add`/`read`/`print` 的实现放此处。

**核心目的**：隐藏实现细节；多 `.cpp` 共用头文件；改 `.cpp` 不必重编全工程（见 [1.5 编译产物](../ch01-getting-started/1.5-类简介.md)）。

## 七、头文件重复包含保护

### 方案 1：`#ifndef` / `#define` / `#endif`（标准）

```cpp
#ifndef SALES_DATA_H
#define SALES_DATA_H
// ...
#endif // SALES_DATA_H
```

- 全平台兼容；宏名冲突会误屏蔽 → 建议 `项目_模块_文件名_H`

### 方案 2：`#pragma once`（编译器扩展）

```cpp
#pragma once
```

- 一行、无宏冲突；GCC/Clang/MSVC 均支持；非 C++ 标准

| 特性 | `#ifndef` | `#pragma once` |
|------|-----------|----------------|
| 标准 | ✅ | 扩展 |
| 适用 | 跨平台库、嵌入式 | 现代本地项目 |

## 八、高频易错点

1. **`struct` 末尾分号不能漏**
2. **类内初值不能用 `()`**，只能用 `=` 或 `{}`
3. **C++11 之前**不支持类内 `= 0`；内置类型若不初始化，**局部对象**可能是脏值
4. 头文件勿写普通函数实现 / 全局变量定义 → `multiple definition`
5. 加了自定义构造后，**聚合 `{}` 三参数初始化失效**，改用对应构造函数
6. 删掉 struct 前的 `add`/`read`/`print` 原型 → 流构造函数等处 **`read` 未声明**
7. 全局变量在 `.h` 只写 `extern`（见 [2.2 extern](./2.2-变量.md)）

---

## 九、背诵极简版

```
Sales_data：bookNo + units_sold=0 + revenue=0.0；struct 末尾 ;
类内初值：= 或 {}，禁止 ()；构造列表 > 类内默认
无自定义构造：合成默认构造 + 聚合 {}
完整版：构造/友元/combine/avg_price；用 Sales_data(s,n,p)
友元：struct 前三行原型 + 内 friend；类外调 add/read/print
.h 声明 .cpp 实现  #ifndef 跨平台  #pragma once 省事
```

## 示例

### 最小可运行（入门三成员，单文件）

复制保存为 `main.cpp`，`g++ -std=c++11 main.cpp -o main` 或 MSVC 等价命令：

```cpp
#include <iostream>
#include <string>

struct Sales_data {
    std::string bookNo;
    unsigned units_sold = 0;
    double revenue = 0.0;
};

int main() {
    Sales_data item1;                                    // "", 0, 0.0
    Sales_data item2{"978-0-201-78345-0", 25, 22.5};    // 聚合初始化

    std::cout << item2.bookNo << " sold " << item2.units_sold
              << ", revenue " << item2.revenue << std::endl;
    return 0;
}
```

### 完整版可运行（单文件，教材接口）

将上文 **第三节** 全部代码 + 下列 `main` 放在同一 `.cpp`：

```cpp
int main() {
    Sales_data item1;
    Sales_data item2("978123456", 5, 39.9);
    print(std::cout, item2) << std::endl;
    return 0;
}
```

编译：`g++ -std=c++11 sales_demo.cpp -o sales_demo`

### 删掉前置声明必报错（复现 `read` 未声明）

见 **第四节「错误示范」**；去掉 struct 前三行后保留 `Sales_data(std::istream& is) { read(is, *this); }` 即可复现。
