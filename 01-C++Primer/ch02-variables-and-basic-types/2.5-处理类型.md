# 2.5 处理类型

## 本节讲什么

针对程序中越来越复杂的类型，介绍简化类型管理的几种方法：使用 `typedef` 和 C++11 的 `using` 定义**类型别名**；使用 **`auto`** 让编译器自动推断变量类型；使用 **`decltype`** 根据表达式推断要定义的变量类型，而不实际计算表达式的值。

## 要点

- **类型别名**：`typedef int wages;` 或 `using wages = int;`（C++11，更清晰）
- **`auto`**：由初始值推断类型，如 `auto i = 42;` → `int`
- **`auto` 引用**：`auto &r = i;` 保留底层 const；`auto *p = &i;` 推断指针
- **`decltype(expr)`**：推断 `expr` 的类型，不执行 `expr`（除 decltype 本身）
- **`decltype((var))`**：加括号得到引用类型，因为 `(var)` 是左值表达式

## 示例

```cpp
#include <iostream>
#include <vector>
int main() {
    // 类型别名
    typedef double wages;
    using salary = double;
    wages hourly = 25.0;
    salary monthly = 5000.0;

    // auto
    auto i = 42;              // int
    auto d = 3.14;            // double
    auto &r = i;              // int&
    const auto ci = i;        // const int

    // decltype
    decltype(i) j = 0;        // int
    decltype(d + i) sum;      // double（不执行加法，只推断类型）
    decltype((i)) k = i;      // int&（括号使表达式为左值）

    std::vector<int> v{1, 2, 3};
    for (auto it = v.begin(); it != v.end(); ++it)
        std::cout << *it << " ";
    std::cout << std::endl;

    return 0;
}
```

## 与 Rust 对照

`typedef` / `using` 与 Rust 的 `type` 同属**类型别名**——只给已有类型起新名字，不改变内存模型。

| C++ | Rust |
|-----|------|
| `using MyInt = int;` | `type MyInt = i32;` |
| `typedef int MyInt;` | `type MyInt = i32;` |

```cpp
using MyInt = int;   // MyInt 与 int 完全等价
```

```rust
type MyInt = i32;    // MyInt 与 i32 完全等价
```

> **不要混淆**：Rust 的 `type` **不对应** [2.3](./2.3-复合类型.md) 里的指针 `*`、引用 `&`；那些对应 Rust 的裸指针和借用。

| 概念 | C++ | Rust |
|------|-----|------|
| 类型别名 | `using` / `typedef` | `type` |
| 内存间接访问 | 指针 `*`、引用 `&` | 裸指针、借用 `&` / `&mut` |
