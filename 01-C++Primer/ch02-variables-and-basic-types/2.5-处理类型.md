# 2.5 处理类型

## 本节讲什么

针对程序中越来越复杂的类型，介绍简化类型管理的几种方法：使用 `typedef` 和 C++11 的 `using` 定义**类型别名**；使用 **`auto`** 让编译器自动推断变量类型；使用 **`decltype`** 根据表达式推断要定义的变量类型，而不实际计算表达式的值。

## 要点

- **类型别名**：优先 `using 别名 = 原类型;`（C++11）；`typedef` 为 C 遗留
- **`auto`**：必须初始化；默认**丢弃顶层 const 和引用**
- **`auto&`**：保留 const/引用，绑定原对象；**`const auto&`** 只读、零拷贝
- **`decltype(expr)`**：只分析类型，**不执行**表达式；完整保留 const/引用
- **`decltype((var))`**：双层括号 → 左值引用

## 一、类型别名：`typedef` / `using`

### 1. `typedef`（C 语言遗留）

```cpp
typedef int wages;
typedef double WageArr[20];   // WageArr 等价 double[20]
typedef char* CharPtr;
```

缺点：顺序反直觉；**不支持模板别名**；复合 const 易踩坑：

```cpp
typedef char* CharPtr;
const CharPtr p = nullptr;   // 等价 char* const p（常量指针），不是 const char*
```

### 2. `using`（C++11 推荐）

```cpp
using wages = int;
using IntVec = std::vector<int>;
using CharPtr = char*;
```

优势：左右对应直观；**支持模板别名**；数组、指针、const 组合更清晰。

```cpp
template<typename T>
using Vec = std::vector<T>;   // typedef 做不到
```

## 二、`auto` 自动类型推导（C++11）

### 基础规则

**必须初始化**；根据**等号右侧**推导；**默认丢弃顶层 const 和引用**。

```cpp
auto i = 42;        // int
auto d = 3.14;      // double
const int ci = 10;
auto x = ci;        // int（顶层 const 被丢弃，拷贝）
```

### `auto&` — 保留引用与底层 const

```cpp
const int ci = 10;
auto &r = ci;       // const int&，保留 const，绑定 ci
int a = 1;
auto &ra = a;       // int&
```

### `auto*` — 指针推导

保留**底层 const**（指向常量的指针）：

```cpp
int val = 5;
auto *p = &val;         // int*
const int cval = 6;
auto *cp = &cval;       // const int*
```

### 三种写法对比

| 写法 | 行为 | 常用场景 |
|------|------|----------|
| `auto` | 值拷贝，丢顶层 const/引用 | 简单局部变量 |
| `auto&` | 绑定左值，保留 const/引用 | 需要改原对象或避免拷贝 |
| `const auto&` | 只读引用，零拷贝 | **范围 for、传参式遍历** |

```cpp
std::vector<std::string> words{"a", "b"};
for (const auto &w : words) { /* 只读，不拷贝 string */ }
```

## 三、`decltype(expr)` 表达式类型推导

**只分析表达式类型，不执行计算**；完整保留 const、引用。

### 规则 1：`decltype(变量名)` — 不加括号

返回变量**声明时的完整类型**：

```cpp
int i = 0;
const int ci = 10;
int &ri = i;

decltype(i) a;       // int
decltype(ci) b;        // const int
decltype(ri) c = i;  // int&（引用必须初始化）
```

### 规则 2：`decltype((变量名))` — 双层括号

`(var)` 是**左值表达式** → 结果为 **`T&`**；原变量有 const 则为 `const T&`：

```cpp
int x = 10;
decltype((x)) rx = x;      // int&

const int cx = 20;
decltype((cx)) rcx = cx;   // const int&
```

### 其他表达式

```cpp
int arr[5];
decltype(arr) arr2;         // int[5]

int num = 1;
decltype(num + 5) res;      // int（算术表达式为右值，无引用）
```

## 四、`auto` vs `decltype` 对比

| 特性 | `auto` | `decltype(expr)` |
|------|--------|------------------|
| 推导依据 | 初始化时的**值** | 任意**表达式** |
| 是否必须初始化 | ✅ 必须 | ❌ 不必（但引用类型须绑定） |
| const/引用 | 默认丢顶层 const、引用 | **完整保留** |
| 括号特殊规则 | 无 | `(var)` → 左值引用 |
| 典型场景 | 迭代器、lambda、简化局部变量 | 模板返回值、精确复刻已有类型 |

## 五、一页速记

```
别名：  using T = ...  优于 typedef
auto：  必须初始化；默认拷贝，丢顶层 const
auto&： 绑左值，保留 const/引用
const auto&：只读 + 零拷贝（遍历首选）

decltype(x)   →  x 的声明类型（含 const/引用）
decltype((x))  →  T& 或 const T&（左值表达式）

typedef 坑：const CharPtr → char* const
```

## 综合示例

```cpp
#include <iostream>
#include <vector>

typedef int wages;
using Wage = int;

int main() {
    auto i = 42;

    const int ci = 100;
    auto &r = ci;           // const int&
    auto *p = &i;           // int*

    decltype(ci) c1 = 200;  // const int
    decltype((ci)) cr = ci; // const int&

    std::vector<int> v{1, 2, 3};
    for (const auto &x : v)
        std::cout << x << " ";
    std::cout << std::endl;

    return 0;
}
```

## 与 Rust 对照

`typedef` / `using` 与 Rust 的 `type` 同属**类型别名**——只给已有类型起新名字，不改变内存模型。

| C++ | Rust |
|-----|------|
| `using MyInt = int;` | `type MyInt = i32;` |
| `typedef int MyInt;` | `type MyInt = i32;` |

> **不要混淆**：Rust 的 `type` **不对应** [2.3](./2.3-复合类型.md) 里的指针、引用；那些对应 Rust 的裸指针和借用。

| 概念 | C++ | Rust |
|------|-----|------|
| 类型别名 | `using` / `typedef` | `type` |
| 类型推断 | `auto` / `decltype` | 部分场景编译器推断 + 显式类型 |

## 示例

```cpp
#include <iostream>
#include <vector>
int main() {
    using salary = double;
    salary monthly = 5000.0;

    auto i = 42;
    auto &r = i;
    decltype((i)) k = i;

    std::vector<int> v{1, 2, 3};
    for (auto it = v.begin(); it != v.end(); ++it)
        std::cout << *it << " ";
    std::cout << std::endl;
    return 0;
}
```
