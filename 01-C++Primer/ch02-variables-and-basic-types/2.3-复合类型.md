# 2.3 复合类型

## 本节讲什么

详细介绍两种基于其他类型定义的复合类型：**引用（Reference）**和**指针（Pointer）**。

- **引用**为对象起了另外一个名字（别名），引用必须被初始化，且无法重新绑定到另外的对象。
- **指针**实现了对其他对象的间接访问，它本身是一个对象，可以被赋值和拷贝。介绍空指针（推荐 C++11 的 `nullptr`）、解引用 `*`、取地址 `&` 以及 `void*` 指针。
- 探讨复杂声明：指向指针的指针、对指针的引用等。

## 要点

- **引用** `int &r = i;`：必须初始化，不能重新绑定，不存在空引用
- **`T&` / `const T&`**：可变引用 vs 常量引用（只读）
- **指针** `int *p = &i;`：可为空，可重新指向其他对象
- **`nullptr`**：空指针字面值，优于 `NULL` 或 `0`
- **解引用** `*p`：访问指针所指对象；**取地址** `&obj`：获取对象地址
- **`void*`**：可存放任意对象地址，但不能直接解引用
- **复杂声明**：从变量名出发，由内向外结合声明符理解，如 `int *&rp` 是对指针的引用

## 为什么 C++ 还要有引用？

已有指针，再引入引用，是为了解决指针在**业务代码**里的常见痛点。

### 指针的缺点

1. **可为空**（`nullptr`）→ 必须判空，漏了可能崩溃（HFT 里直接宕机）
2. **可随意改指向** → 逻辑容易乱
3. **语法繁琐** → `*p` 解引用、`*`/`&` 混用
4. **可做算术**（`p++`）→ 容易越界

```cpp
int a = 1, b = 2;
int *p = &a;
p = &b;   // 随时换指向
```

### 引用：受约束的「安全版指针」

编译器实现上，引用常类似 **`T* const`**（指向 `T` 的**常量指针**——地址绑死不能改）：

| 特性 | 指针 `int*` | 引用 `int&` |
|------|-------------|-------------|
| 可为空 | ✅ `nullptr` | ❌ 必须绑定对象 |
| 改指向 | ✅ `p = &other` | ❌ 绑定后不能换 |
| 使用语法 | `*p` | 直接当变量用 |
| 指针运算 | ✅ | ❌ |

```cpp
int a = 10;
int &ref = a;
ref = 20;              // 直接改 a，不用 *

// 底层直觉（概念模型，不是你要写的代码）：
// int *const p = &a;  // 地址固定，通过 *p 访问
```

**一句话**：引用 = **受限制、更安全、语法更简洁**的间接访问；日常业务优先引用，底层内存/DPDK 才用裸指针。

### 取舍标准（HFT 编码习惯）

**优先用引用：**

- 函数传参、避免拷贝：`void f(const OrderBook &book);`
- 运算符重载（语法要求）：`operator+`、`operator<<` 等

**必须用指针：**

- 后期要**改变指向**
- 对象**可能不存在**（可选）
- 数组、内核、DPDK 等需要**地址偏移**

## 引用：`T&` 与 `const T&`

### 1. 可变引用（可修改原对象）

```cpp
int a = 10;
int &ref = a;
ref = 20;        // ✅ 通过 ref 修改 a
// a 现在是 20
```

- `int&`：绑定原变量，**可读可写**

### 2. 常量引用（只读）

```cpp
int a = 10;
const int &ref = a;
// ref = 20;     // ❌ 编译错误，不能通过 ref 修改
std::cout << ref; // ✅ 只读
```

- `const int&`：**只能读**，不能通过引用改原对象
- 可绑定到字面量：`const int &r = 42;`（普通 `int&` 不能绑字面量）

### 3. 引用的硬性规则（C++ 独有）

1. **必须初始化**，不能空着
2. **绑定后不能改绑**到其他对象（Rust 的引用可以重新绑定）
3. 不存在「空引用」

### 4. 工程习惯（HFT / 日常 C++）

```cpp
// 传参优先：不拷贝 + 只读
void process(const std::string &s);

// 需要改入参时才用可变引用
void adjust(int &value);
```

| 场景 | 用法 |
|------|------|
| 只读、避免拷贝 | `const T&` |
| 需要修改原对象 | `T&` |
| 可选/可为空 | 用指针或 `std::optional`（指针章节） |

## 指针：`int*` 与 `const`（和引用放一起记）

| 写法 | 读法 | 谁能改 |
|------|------|--------|
| `int*` / `int *p` | 普通指针 | 指针可变指向；可通过 `*p` 改对象 |
| `const int*` / `int const *p` | 指向常量的指针 | **不能**通过 `*p` 改对象；`p` 可改指向 |
| `int* const p` | 常量指针 | 可通过 `*p` 改对象；**不能**改 `p` 的指向 |
| `const int* const p` | 都 const | 指向和对象都不改 |

```cpp
int i = 0, j = 1;
const int *p = &i;   // *p 不可改，p 可指向 j
p = &j;              // ✅
// *p = 2;           // ❌

int *const cp = &i;  // cp 不可改指向，*cp 可改
*cp = 2;             // ✅
// cp = &j;          // ❌
```

**和引用的对应：**

| 引用 | 近似指针含义 |
|------|--------------|
| `int&` | 像绑死的 `int*`，必须有效，可读写 `*p` |
| `const int&` | 像 `const int*`，只读，不能 `*p = ...` |

引用不能重新绑定 ≈ 没有「改指向」这回事；指针可以 `p = &other`。

## 与 Rust 对照（横向一览）

| | C++ | Rust |
|---|-----|------|
| **只读间接访问** | `const T&` | `&T`（借用检查） |
| **可变间接访问** | `T&` | `&mut T`（借用检查） |
| **裸指针** | `T*`、`const T*` | `*const T`、`*mut T`（多在 `unsafe`） |
| **编译期安全** | ❌ 靠程序员 + `const` | ✅ 借用规则强制 |

- C++：`T&` / `const T&` ≈ **安全版**用法；`T*` ≈ **自由、风险高**
- Rust：`&T` / `&mut T` ≈ C++ 引用 **再叠加** 编译期借用检查；裸指针脱离规则，在 `unsafe` 里用

**关键区别：**

- Rust：**编译期借用检查**，`&T` 与 `&mut T` 不能同时活跃（同一数据）
- C++：**没有**强制借用检查；`T&` 和 `const T&` 可同时存在——多线程下可能**数据竞争**，HFT 要自己管同步

简单记：**C++ 有引用和裸指针两层；Rust 把引用管得更严，裸指针放进 unsafe。**

## 示例

```cpp
#include <iostream>
#include <string>

void read_only(const std::string &s) {
    std::cout << s << std::endl;
}

void bump(int &n) {
    ++n;
}

int main() {
    int ival = 42;

    int &refVal = ival;
    refVal = 1024;
    std::cout << ival << std::endl;  // 1024

    const int &cref = ival;
    std::cout << cref << std::endl;

    int *p = &ival;
    *p = 0;
    std::cout << ival << std::endl;  // 0

    const int *pc = &ival;           // 指向常量：不能 *pc 改值
    int *const cp = &ival;           // 常量指针：cp 不能改指向

    read_only("hello");
    bump(ival);

    return 0;
}
```
