# 3.2 标准库类型 string

## 本节讲什么

`std::string` 是 C++ 处理文本的**核心类**，也是后面写 HFT 策略、处理**订单 / 行情 / 日志**文本的基础。本节重点：**初始化方式**、**读写操作**、**内存管理逻辑**，以及和 **C 风格字符串**的本质区别——`string` **自动管理内存**，不必手动 `malloc` / `free`，能避开大量泄漏坑。

## 要点

- **`#include <string>`**；可变长，**RAII 自动释放**内存
- **初始化**：默认空串、拷贝、字面值、`string(n, c)` 重复字符
- **读写**：`cin >> s` 读单词；`getline(cin, s)` 读整行；`cout << s` 输出
- **操作**：`+` / `+=` 拼接、`==` 比较、`size()` / `empty()` / `[]` / `substr`
- **vs C 字符串**：`char*` + 手动分配 vs `string` 随内容增长自动扩缩
- **HFT**：FIX 报文解析、日志拼接、合约代码 / 订单号处理（见文末拓展）

## 一、定义与初始化

```cpp
#include <string>

std::string s1;                    // 空串 ""
std::string s2 = "hello";          // 字面值
std::string s3(s2);                // 拷贝
std::string s4(10, 'c');           // "cccccccccc"
std::string s5 = std::string(5, 'x');  // "xxxxx"
```

| 写法 | 结果 |
|------|------|
| `string s;` | 默认构造，空串 |
| `string s = "abc";` | 从 C 风格字面量拷贝内容 |
| `string s(other);` | 深拷贝 `other` 的字符序列 |
| `string s(n, c);` | `n` 个字符 `c` |

列表初始化（C++11）：

```cpp
std::string s{"IF2406"};   // 与 = "IF2406" 等价
```

## 二、读写操作

### 输出

```cpp
std::string msg = "order sent";
std::cout << msg << std::endl;
```

### 输入：`>>` vs `getline`

**统一规则**：`>>` 把**任意数量、连续**的空白符（空格、换行 `\n`、制表符 `\t` 等）都当作**分隔符**——开头、中间、结尾的空白都会被**跳过**，只依次取出非空白字段。`getline` **逻辑相反**：不跳过开头/中间空白，只在**换行**处停止，换行前的字符（含空格、Tab）**原样**存入 `string`。

| 方式 | 停止条件 | 空白处理 | 典型场景 |
|------|----------|----------|----------|
| `cin >> s` | 遇**任意空白**即停 | **跳过**所有连续空白（前/中/后当分隔符） | 结构化字段：订单号、价格、数量 |
| `getline(cin, s)` | 仅遇**换行 `\n`** 才停 | **保留**开头与中间空格、Tab | 带空格备注、整行日志 |

#### 同一行输入，两种读法对比

假设键盘输入（`·` 表示空格）：

```
···hello···world⏎
（3 个前导空格 + hello + 3 个中间空格 + world + 回车）
```

| 读法 | 结果 |
|------|------|
| `cin >> w1 >> w2` | `w1="hello"`, `w2="world"`（前后、中间空白全当分隔符忽略） |
| `getline(cin, line)` | `line="···hello···world"`（前导与中间空格**全部保留**） |

简单输入 `"hello world"` + 回车，`cin >> word1 >> word2` 同样得到 `"hello"` / `"world"`。

#### 常见误解：`cin >>` 是不是不按回车就会一直读？

**不是。** 碰到**空白符**就停；回车只是产生换行空白的一种。剩余内容留在**输入缓冲区**，供下一次 `>>` 使用：

```
键盘输入:   [空格空格] hello world [回车]
            ─────────────────────────────► 输入缓冲区

第 1 次 cin >> word1:
  跳过开头所有空白 → 读 "hello" → 遇空格停止
  缓冲区剩余: " world\n"

第 2 次 cin >> word2:
  跳过空白 → 读 "world" → 遇换行停止
```

#### HFT：结构化订单流 vs 带空格备注

**`cin >>` —— 逐字段拆分**（每行空格分隔，中间多个空格也会自动合并跳过分隔）：

```cpp
// 输入一行:  OID1001  3750.5  10   （字段间可有多个空格）
std::string order_id;
double      price = 0;
int         qty   = 0;

std::cin >> order_id >> price >> qty;
// order_id == "OID1001", price == 3750.5, qty == 10
// 不必自己写循环跳过多个空格
```

适合：测试单、回放文件里**固定列**的订单流（订单号 / 价格 / 数量）。

**`getline` —— 保留空格的全文**：

```cpp
// 备注: "urgent  buy IF2406"（含多个空格）
std::string remark;
std::getline(std::cin, remark);   // 整行原样，直到回车
```

适合：带格式的**日志行**、带空格的**订单备注**、人工可读说明。

```
结构化字段（空格当分隔）  →  cin >>
保留空格的全行文本        →  getline
```

#### `>>` 与 `getline` 混用注意

`cin >> n` 后若直接 `getline(cin, line)`，可能读到**空行**——因为 `>>` 留下行尾的 `\n` 在缓冲区里。常见写法：

```cpp
int qty;
std::string remark;
std::cin >> qty;
std::cin.ignore();              // 吃掉 >> 留下的换行
std::getline(std::cin, remark);
```

```cpp
std::string word, line;
std::cin >> word;
std::getline(std::cin, line);   // 读同一行剩余，或下一整行（视缓冲区而定）
```

### 拼接与比较

```cpp
std::string a = "buy", b = "IF2406";
std::string c = a + " " + b;   // "buy IF2406"
a += " done";

if (a == b) { /* ... */ }
if (a.size() > 0) { /* 或 !a.empty() */ }
```

## 三、内存管理：string vs C 风格字符串

### C 风格：`char*` 是什么？

`char *`（或 `char[]`）本质是**普通指针**，指向字符数组的**首元素**；字符串以 **`'\0'`** 结束，长度靠扫到 `\0` 才知道——**没有**内建的 `size()`：

```cpp
const char* msg = "FIX.4.2";   // 指向只读字符序列首字符
char buf[64] = "order";        // 固定数组，要自己保证 '\0' 和容量

// 拼接、变长都要手动 malloc/realloc、strcpy/strcat，极易越界或忘 '\0'
```

读网关旧 API 时常见 **`const char*`**（只读视图）——对应 const 里的 **指向常量的指针** `const char *p`：不能通过 `p` 改内容，但 `p` 可改指向别的缓冲区。

### string 封装了什么？

`std::string` 把**指针、数组、长度、`'\0'`、内存分配**等细节收成类，你只调接口：

| 你关心的 | C 风格要自己管 | `string` |
|----------|---------------|----------|
| 长度 | 扫 `\0` 或另记计数 | `size()` / `empty()` |
| 追加 | `strcat` + 缓冲够大 | `append()` / `+=` |
| 内存 | `malloc` / `free` | 析构自动释放 |
| 结束符 | 手动保证 `\0` | 内部维护 |

```cpp
std::string fix = "8=FIX.4.2|";
fix.append("35=D|");           // 不用算 strlen、不用怕 '\0'
std::cout << fix.size();      // 字符个数，不含你操心的 '\0' 细节
```

**HFT 低 bug 率**：业务字符串优先 `string`，少碰裸 `char*` 的手动内存与 `\0`；必须对接 C API 时用 `c_str()` **临时**过渡。

| | **`std::string`** | **C 风格 `char*` / `char[]`** |
|---|-------------------|-------------------------------|
| 长度 | **可变**，随内容自动增长/收缩 | 固定缓冲或裸指针，易越界 |
| 内存 | 构造时分配、析构时**自动释放**（RAII） | `malloc` / `new[]` 须配对 `free` / `delete[]` |
| 拼接 | `+` / `+=` 自动重新分配 | `strcat` 等须保证目标缓冲足够大 |
| 安全性 | 边界由 `size()` 管理；`at(i)` 越界检查 | 无长度信息，**缓冲区溢出**常见 |

```cpp
// ❌ C 风格：易忘释放或越界
char* p = new char[100];
// ... 若异常路径未 delete[] → 泄漏

// ✅ string：作用域结束自动析构
{
    std::string fix = "8=FIX.4.2|35=D|";
}   // 内存自动回收
```

**与 C 接口互操作**（读旧 API、写 C 函数时）：

```cpp
std::string s = "hello";
const char* c = s.c_str();   // 只读 C 风格视图；s 改内容后可能失效
// s[0] = 'H';               // 再调 c_str() 前勿假设 c 仍有效
```

HFT 新代码优先 `string`；底层网关若只给 `const char*`，可立刻包一层 `std::string msg(ptr);` 再处理。

## 四、`cctype`：按字符处理

`#include <cctype>`，对**单个字符**分类 / 转换（参数类型为 `int`，通常传 `unsigned char` 避免负值 UB）：

| 函数 | 含义 |
|------|------|
| `isalpha(c)` | 字母 |
| `isdigit(c)` | 数字 |
| `isspace(c)` | 空白 |
| `isalnum(c)` | 字母或数字 |
| `toupper(c)` / `tolower(c)` | 大小写转换 |

```cpp
for (unsigned char c : symbol) {
    if (std::isdigit(c))
        /* 数字段 */;
}
```

## 五、HFT 场景小拓展

### 1. FIX 报文（简化示意）

FIX 是交易所常见的**键值对 + 分隔符**文本（`|` 或 `\001` SOH）。`string` 适合整段持有报文，再 `find` / `substr` 取字段：

```cpp
#include <string>

// 简化：用 '|' 代替 SOH
std::string fix = "8=FIX.4.2|35=D|11=OID1001|55=IF2406|38=10|";

std::string tag_11;   // ClOrdID
auto pos = fix.find("11=");
if (pos != std::string::npos) {
    auto start = pos + 3;
    auto end = fix.find('|', start);
    tag_11 = fix.substr(start, end - start);   // "OID1001"
}
```

生产环境会用专业 FIX 库；这里说明 **`string` 是报文容器 + 字段切分的基础**。

### 2. 日志拼接

```cpp
std::string log;
log += "[";
log += std::to_string(order_id);   // C++11
log += "] send ";
log += symbol;
log += " qty=";
log += std::to_string(qty);
// 或一次拼接："[123] send IF2406 qty=10"
```

热路径上若频繁拼接，可考虑 `reserve()` 预分配减少重分配，或固定缓冲 + 长度（进阶优化）；入门阶段 **`string` + `+=` 清晰安全**即可。

### 3. 合约 / 订单文本

- 合约代码：`"IF2406"`、`"600519.SH"` —— `string` 比较、哈希、作 map 键
- 与 [2.6 Sales_data](../ch02-variables-and-basic-types/2.6.1-Sales_data入门与聚合.md) 的 `bookNo` 同类：**业务文本字段用 `string`，数值用算术类型**

## 六、背诵极简版

```
string：RAII；封装指针/长度/'\0'，用 size/append
C char*：指首字符，手动 '\0' 与内存；读 API 常见 const char*
const int*p 指向常量；int*const p 常量指针（见 2.4.1）
读：>> 连续空白当分隔符，逐字段；getline 保留空格读到换行
HFT：cin>> 拆订单号/价/量；getline 读备注与日志
拼接 +/+=  比较 ==  长度 size/empty
vs char*：string 安全；C 接口用 c_str() 过渡
HFT：FIX 报文容器、日志拼接、合约/订单号
```

## 示例

```cpp
#include <iostream>
#include <string>
#include <cctype>

int main() {
    std::string s1 = "hello", s2;
    s2 = s1;
    s1 = "world";
    std::cout << s1 << " " << s2 << std::endl;  // world hello

    std::string greeting = s1 + ", " + s2;
    std::cout << greeting << std::endl;

    std::string line;
    std::getline(std::cin, line);

    for (unsigned char c : s1) {
        if (std::isalpha(c))
            std::cout << static_cast<char>(std::toupper(c));
    }
    std::cout << std::endl;

    return 0;
}
```
