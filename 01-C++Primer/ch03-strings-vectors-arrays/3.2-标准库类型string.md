# 3.2 标准库类型 string

## 本节讲什么

`std::string` 是 C++ 处理文本的**核心类**，也是后面写 HFT 策略、处理**订单 / 行情 / 日志**文本的基础。本节重点：**初始化方式**、**读写操作**、**内存管理逻辑**，以及和 **C 风格字符串**的本质区别——`string` **自动管理内存**，不必手动 `malloc` / `free`，能避开大量泄漏坑。

## 要点

- **`#include <string>`**；可变长，**RAII 自动释放**内存
- **初始化**：默认空串、拷贝、字面值、`string(n, c)` 重复字符
- **读写**：`cin >> s` 读单词；`getline(cin, s)` 读整行；`cout << s` 输出
- **操作**：`+` / `+=` 拼接、`==` 比较、`size()` / `empty()` / `[]` / `substr`
- **vs C 字符串**：`char*` + 手动分配 vs `string` 随内容增长自动扩缩
- **HFT**：FIX 报文解析、日志拼接、合约代码 / 订单号处理（见文末拓展）
- **`<cctype>`**：`isalpha`/`isdigit` 等单字符校验；参数须 **`unsigned char`**

## 一、定义与初始化

```cpp
#include <string>

std::string s1;                    // 空串 ""
std::string s2 = "hello";          // 字面值
std::string s3(s2);                // 拷贝
std::string s4(10, 'c');           // "cccccccccc"
std::string s5 = std::string(5, 'x');  // "xxxxx"
```

| 写法 | 结果 |
|------|------|
| `string s;` | 默认构造，空串 |
| `string s = "abc";` | 从 C 风格字面量拷贝内容 |
| `string s(other);` | 深拷贝 `other` 的字符序列 |
| `string s(n, c);` | `n` 个字符 `c` |

列表初始化（C++11）：

```cpp
std::string s{"IF2406"};   // 与 = "IF2406" 等价
```

## 二、读写操作

### 输出

```cpp
std::string msg = "order sent";
std::cout << msg << std::endl;
```

### 输入：`>>` vs `getline`

**统一规则**：`>>` 把**任意数量、连续**的空白符（空格、换行 `\n`、制表符 `\t` 等）都当作**分隔符**——开头、中间、结尾的空白都会被**跳过**，只依次取出非空白字段。`getline` **逻辑相反**：不跳过开头/中间空白，只在**换行**处停止，换行前的字符（含空格、Tab）**原样**存入 `string`。

| 方式 | 停止条件 | 空白处理 | 典型场景 |
|------|----------|----------|----------|
| `cin >> s` | 遇**任意空白**即停 | **跳过**所有连续空白（前/中/后当分隔符） | 结构化字段：订单号、价格、数量 |
| `getline(cin, s)` | 仅遇**换行 `\n`** 才停 | **保留**开头与中间空格、Tab | 带空格备注、整行日志 |

#### 同一行输入，两种读法对比

假设键盘输入（`·` 表示空格）：

```
···hello···world⏎
（3 个前导空格 + hello + 3 个中间空格 + world + 回车）
```

| 读法 | 结果 |
|------|------|
| `cin >> w1 >> w2` | `w1="hello"`, `w2="world"`（前后、中间空白全当分隔符忽略） |
| `getline(cin, line)` | `line="···hello···world"`（前导与中间空格**全部保留**） |

简单输入 `"hello world"` + 回车，`cin >> word1 >> word2` 同样得到 `"hello"` / `"world"`。

#### 常见误解：`cin >>` 是不是不按回车就会一直读？

**不是。** 碰到**空白符**就停；回车只是产生换行空白的一种。剩余内容留在**输入缓冲区**，供下一次 `>>` 使用：

```
键盘输入:   [空格空格] hello world [回车]
            ─────────────────────────────► 输入缓冲区

第 1 次 cin >> word1:
  跳过开头所有空白 → 读 "hello" → 遇空格停止
  缓冲区剩余: " world\n"

第 2 次 cin >> word2:
  跳过空白 → 读 "world" → 遇换行停止
```

#### HFT：结构化订单流 vs 带空格备注

**`cin >>` —— 逐字段拆分**（每行空格分隔，中间多个空格也会自动合并跳过分隔）：

```cpp
// 输入一行:  OID1001  3750.5  10   （字段间可有多个空格）
std::string order_id;
double      price = 0;
int         qty   = 0;

std::cin >> order_id >> price >> qty;
// order_id == "OID1001", price == 3750.5, qty == 10
// 不必自己写循环跳过多个空格
```

适合：测试单、回放文件里**固定列**的订单流（订单号 / 价格 / 数量）。

**`getline` —— 保留空格的全文**：

```cpp
// 备注: "urgent  buy IF2406"（含多个空格）
std::string remark;
std::getline(std::cin, remark);   // 整行原样，直到回车
```

适合：带格式的**日志行**、带空格的**订单备注**、人工可读说明。

```
结构化字段（空格当分隔）  →  cin >>
保留空格的全行文本        →  getline
```

#### `>>` 与 `getline` 混用注意

`cin >> n` 后若直接 `getline(cin, line)`，可能读到**空行**——因为 `>>` 留下行尾的 `\n` 在缓冲区里。常见写法：

```cpp
int qty;
std::string remark;
std::cin >> qty;
std::cin.ignore();              // 吃掉 >> 留下的换行
std::getline(std::cin, remark);
```

```cpp
std::string word, line;
std::cin >> word;
std::getline(std::cin, line);   // 读同一行剩余，或下一整行（视缓冲区而定）
```

### 拼接与比较

```cpp
std::string a = "buy", b = "IF2406";
std::string c = a + " " + b;   // "buy IF2406"
a += " done";

if (a == b) { /* ... */ }
if (a.size() > 0) { /* 或 !a.empty() */ }
```

#### `s1 + s2` 的内存细节（临时对象 + 拷贝/移动）

```cpp
std::string s1 = "hello", s2 = "world";
std::string s3 = s1 + s2;
```

**逐步拆解：**

1. **`s1 + s2`**：`operator+` 生成一个**临时 `string`**，申请一块能装下 `s1+s2` 的新内存，把两边字符**拷贝**进去  
2. **初始化 `s3`**：把临时对象的内容**拷贝或移动**进 `s3`（C++11 起对临时对象多为 **move**，不是必做深拷贝，但临时仍要先分配一块）  
3. **语句结束**：临时对象**马上析构**，释放其内存  

| 阶段 | 对象 / 内存 |
|------|-------------|
| 语句前 | `s1`、`s2`（**2 个**具名对象） |
| 算 `s1 + s2` | **+1 临时** `string`（新分配 + 拷贝 s1、s2 内容） |
| 赋给 `s3` | `s3` 再分配（或 move 接管）临时的内容 |
| 语句后 | 临时析构；剩 `s1`、`s2`、`s3`（**3 个**具名对象） |

**至少 1 个临时 + 多次分配/拷贝**（C++11 前常为两次拷贝；现代编译器常 move 减少一次字符拷贝，但**临时对象的分配/析构**仍在）。

链式 `s1 + " " + s2` → **多个临时**，每个 `+` 都可能分配+释放：

```cpp
std::string c = s1 + " " + s2;
// s1+" " → 临时1；临时1+s2 → 临时2；再 move/拷贝进 c；临时1、2 析构
```

#### HFT：少用在热路径上的 `+`，改用 `reserve` + `append`

频繁 `+` 拼接的问题：**临时对象分配/析构** → 延迟抖动、**内存碎片**。

```cpp
// ❌ 热路径：多次临时 + 多次分配
std::string s3 = s1 + s2;

// ✅ 更好：先预留，再一次 append（通常只扩一次容，无临时 string）
std::string s3;
s3.reserve(s1.size() + s2.size());
s3.append(s1).append(s2);
// 或：s3 = s1; s3.append(s2); 若 s3 为空且 s1 不太大也可接受
```

| 方式 | 临时 `string` | 典型分配次数（粗略） |
|------|---------------|---------------------|
| `s3 = s1 + s2` | 至少 1 个 | 临时 1 次 + `s3` 1 次（move 后临时释放） |
| `reserve` + `append` | **无** | 往往 **1 次**（预留足够时） |

传参时用 **`const std::string&`** 避免无谓拷贝（见 [2.3.1 引用](../ch02-variables-and-basic-types/2.3.1-引用.md)）。

#### 和 Rust 的对比

- **Rust**：`+` 也涉及所有权转移/新分配，但编译器 + 移动语义在编译期**减少冗余临时**（如 `format!`、预容量 `String::with_capacity`）  
- **C++**：靠 **`reserve` + `append`**、**移动**、**引用传参** 手动做到接近效率；没有借用检查，但 HFT 里这是常见写法  

**`s1 += s2`**：不新建第三个具名对象，在 **`s1` 上**扩容并追加（必要时内部重分配一次），一般比 `s3 = s1 + s2` 省临时对象。

## 三、内存管理：string vs C 风格字符串

### C 过程式 vs C++ 封装：两种思维

| | **C 风格字符串** | **`std::string`** |
|---|------------------|-------------------|
| 思维 | **裸内存 + 手动管理**（过程式） | **封装内存 + 自动管理**（面向对象 / RAII） |
| 长度 | 自己 `strlen` 或另记 | `size()` |
| 分配/释放 | `malloc` / `free`，易漏 | 构造/析构自动管 |
| 结束符 | 时刻记得 `'\0'`，防越界 | 类内部维护 |
| 操作 | `strcpy` / `strcat` | `append`、`+`（运算符重载，代码更简洁） |

C++ 相对 C 的升级：把「算长度、分配释放、加 `\0`」收成**成员函数 + 运算符**，你只调接口；业务代码更安全。

到了 **HFT 极致性能**场景，有时又会**绕回裸内存布局**——见下文「固定缓冲 + RAII」折中。

### C 风格：`char*` 是什么？

`char *`（或 `char[]`）本质是**普通指针**，指向字符数组的**首元素**；字符串以 **`'\0'`** 结束，长度靠扫到 `\0` 才知道——**没有**内建的 `size()`。

**声明风格**：很多人写 `char* p`，但 `char* p, q;` 只会把 **`p`** 声明成指针，**`q` 仍是 `char`**——经典坑。严谨写法 **`char *p, *q;`**（见 [2.4.2 声明语法](../ch02-variables-and-basic-types/2.4.2-const口诀与声明语法.md)）。

```cpp
const char *msg = "FIX.4.2";   // 推荐：* 靠变量名
char buf[64] = "order";

// 拼接、变长都要手动 malloc/realloc、strcpy/strcat，极易越界或忘 '\0'
```

读网关旧 API 时常见 **`const char*`**（只读视图）——对应 const 里的 **指向常量的指针** `const char *p`：不能通过 `p` 改内容，但 `p` 可改指向别的缓冲区。

### string 封装了什么？

`std::string` 把**指针、数组、长度、`'\0'`、内存分配**等细节收成类，你只调接口：

| 你关心的 | C 风格要自己管 | `string` |
|----------|---------------|----------|
| 长度 | 扫 `\0` 或另记计数 | `size()` / `empty()` |
| 追加 | `strcat` + 缓冲够大 | `append()` / `+=` |
| 内存 | `malloc` / `free` | 析构自动释放 |
| 结束符 | 手动保证 `\0` | 内部维护 |

```cpp
std::string fix = "8=FIX.4.2|";
fix.append("35=D|");           // 不用算 strlen、不用怕 '\0'
std::cout << fix.size();      // 字符个数，不含你操心的 '\0' 细节
```

**HFT 低 bug 率**：业务字符串优先 `string`，少碰裸 `char*` 的手动内存与 `\0`；必须对接 C API 时用 `c_str()` **临时**过渡。

### RAII、自动扩缩 vs Rust 所有权

**`string` 拥有**自己堆上的字符缓冲区（类似 Rust 的 `String` 拥有堆数据）：

- **变长**：`append` / `+=` / `+` 时，内容变长则**自动重新分配**更大缓冲；变短后**可能**缩小容量（实现相关，不保证立刻 shrink）
- **析构**：`string` 离开作用域 → 析构函数**自动 `delete[]` 内部缓冲** → 不用手写 `free`

这叫 **RAII**（资源获取即初始化）：**构造管分配，析构管释放**。

| | **`std::string`（C++）** | **Rust `String`** |
|---|-------------------------|-------------------|
| 谁管内存 | 对象**拥有**堆缓冲；析构释放 | **所有权**；`drop` 时释放 |
| 自动释放 | 作用域结束析构 | 离开作用域 `drop` |
| 编译期防错 | ❌ 无借用检查；仍有悬垂引用、`char*` 坑 | ✅ 借用检查器 |
| 拼接 | `+` 可能产生**临时** `string` | 移动/所有权，编译期优化更多 |

**相似点**：都是「对象活着就占资源，对象销毁就释放资源」，HFT 里比裸 `char*` 少泄漏。

**不同点**：C++ **没有** Rust 那套所有权 / 借用规则；`string` 仍可与裸指针、悬垂 `string&` 组合出 bug——RAII 只保证 **`string` 自己**析构时释放**自己**的缓冲，不保证你的指针仍有效。

```cpp
std::string s = "order";
const char* raw = s.c_str();
s += " extra";     // s 可能重分配 → raw 悬垂，危险
```

| | **`std::string`** | **C 风格 `char*` / `char[]`** |
|---|-------------------|-------------------------------|
| 长度 | **可变**，随内容自动增长/收缩 | 固定缓冲或裸指针，易越界 |
| 内存 | 构造时分配、析构时**自动释放**（RAII） | `malloc` / `new[]` 须配对 `free` / `delete[]` |
| 拼接 | `+` / `+=` 自动重新分配 | `strcat` 等须保证目标缓冲足够大 |
| 安全性 | 边界由 `size()` 管理；`at(i)` 越界检查 | 无长度信息，**缓冲区溢出**常见 |

```cpp
// ❌ C 风格：易忘释放或越界
char* p = new char[100];
// ... 若异常路径未 delete[] → 泄漏

// ✅ string：作用域结束自动析构
{
    std::string fix = "8=FIX.4.2|35=D|";
}   // 内存自动回收
```

**与 C 接口互操作**（读旧 API、写 C 函数时）：

```cpp
std::string s = "hello";
const char* c = s.c_str();   // 只读 C 风格视图；s 改内容后可能失效
// s[0] = 'H';               // 再调 c_str() 前勿假设 c 仍有效
```

HFT 新代码优先 `string`；底层网关若只给 `const char*`，可立刻包一层 `std::string msg(ptr);` 再处理。

### HFT 折中：固定长度缓冲 + C++ RAII

热路径（高频行情、合约码、短订单号）上，`string` 的**动态分配**可能带来延迟和碎片。常见折中：

- **内存布局用 C 的思路**：栈上或预分配好的 **固定长度 `char` 数组**，大小在编译期或启动时定死，**热路径零 heap 分配**
- **生命周期用 C++ 的思路**：用 **RAII 类/结构体** 包住这块缓冲，析构时自动清理（栈数组本身无需 free；若类内 `new[]` 则析构里 `delete[]`）

```cpp
// 栈上固定缓冲：类似 C，但放在 struct 里便于传递、封装
struct TickSymbol {
    char data[16]{};   // 合约码，如 "IF2406"，栈分配、无动态 alloc

    const char* c_str() const { return data; }
    // 写入时仍须自己保证不越界、保留 '\0'
};

// 或 std::array<char, 16> + 长度字段，仍无 string 的 realloc
#include <array>
struct FastSymbol {
    std::array<char, 16> buf{};
    uint8_t len = 0;
};
```

| 场景 | 推荐 |
|------|------|
| 一般业务、FIX 解析、日志 | `std::string`（安全、可变长） |
| 热路径、长度有上界、极高频 | 固定 `char[]` / `std::array<char,N>` |
| 既要固定缓冲又要异常安全释放堆内存 | 自定义 RAII 类（构造分配、析构释放） |

**结合两者优点**：**C 的内存布局保证性能**，**C++ 的 RAII/类封装保证释放逻辑不遗漏**——不是退回纯 C，而是在关键路径上**有控制地**少用 `string` 的动态性。

## 四、`<cctype>`：单字符判断与转换

`<cctype>` 是 C++ **继承自 C** 的字符处理头文件（C 里 `<ctype.h>`，C++ 用 `<cctype>` 且函数在 **`std` 命名空间**）。**`#include <cctype>`** 在预处理阶段把函数**声明**粘贴进当前文件（见 [2.2.4 `#include` 本质](../ch02-variables-and-basic-types/2.2.4-extern与头文件分工.md)）。

| 函数 | 含义 | HFT 示例用途 |
|------|------|--------------|
| `isalpha(c)` | 是否字母 | 合约代码是否全字母段 |
| `isdigit(c)` | 是否数字 | 价格、数量字段是否全数字 |
| `isalnum(c)` | 字母或数字 | 通用 token 校验 |
| `isspace(c)` | 是否空白 | 跳过分隔符、检查非法空白 |
| `toupper(c)` / `tolower(c)` | 大小写转换 | 统一合约码大小写 |

### 参数必须是 `unsigned char` 转成 `int`

标准规定这些函数的参数类型是 **`int`**，且必须是 **`unsigned char` 的值**或 **`EOF`**。若直接把可能为负的 **`char`** 传入，在 `char` 为 signed 且值为 `0x80`～`FF` 的平台上是**未定义行为**。

```cpp
// ❌ 危险：char 可能为 signed
char ch = data[i];
if (std::isdigit(ch)) { /* ... */ }

// ✅ 正确：先转成 unsigned char
unsigned char uch = static_cast<unsigned char>(data[i]);
if (std::isdigit(uch)) { /* ... */ }

// ✅ 遍历 string 时推荐
for (unsigned char c : symbol) {
    if (!std::isalnum(c))
        /* 非法字符 */;
}
```

这是写底层 / 报文校验代码时容易忽略的细节。

### HFT：快速校验报文字段

从订单报文、FIX 字段里取出子串后，用 `<cctype>` 做**轻量格式检查**（正式环境还有完整 schema 校验）：

```cpp
#include <cctype>
#include <string>

bool all_alpha(const std::string& s) {
    for (unsigned char c : s)
        if (!std::isalpha(c)) return false;
    return !s.empty();
}

bool all_digit(const std::string& s) {
    for (unsigned char c : s)
        if (!std::isdigit(c)) return false;
    return !s.empty();
}

// 用法示意
std::string symbol = "IF";      // 合约代码段
std::string price  = "3750";    // 价格字段
// all_alpha(symbol), all_digit(price)
```

比手写 `c >= '0' && c <= '9'` 更易读；实现通常经库优化，且覆盖 locale 相关边界（纯 ASCII 报文场景下与手写等价）。

```cpp
// 统一转大写合约码
std::string upper_symbol = symbol;
for (char& ch : upper_symbol)
    ch = static_cast<char>(std::toupper(static_cast<unsigned char>(ch)));
```

## 五、HFT 场景小拓展

### 1. FIX 报文（简化示意）

FIX 是交易所常见的**键值对 + 分隔符**文本（`|` 或 `\001` SOH）。`string` 适合整段持有报文，再 `find` / `substr` 取字段：

```cpp
#include <string>

// 简化：用 '|' 代替 SOH
std::string fix = "8=FIX.4.2|35=D|11=OID1001|55=IF2406|38=10|";

std::string tag_11;   // ClOrdID
auto pos = fix.find("11=");
if (pos != std::string::npos) {
    auto start = pos + 3;
    auto end = fix.find('|', start);
    tag_11 = fix.substr(start, end - start);   // "OID1001"
}
```

生产环境会用专业 FIX 库；这里说明 **`string` 是报文容器 + 字段切分的基础**。

```cpp
std::string log;
```

## 六、背诵极简版

```
C 字符串：裸内存+手动；string：封装+RAII+运算符
HFT 一般 string；热路径固定 char[N]/array + RAII 折中
HFT 热路径：reserve+append 代替 +；少临时、少碎片
s3=s1+s2：至少 1 临时 + 多次分配；+= 改 s1 更省
C char*：指首字符；声明用 char *p（勿写 char* p,q 坑）
const int*p 指向常量；int*const p 常量指针（见 2.4.1）
读：>> 连续空白当分隔符，逐字段；getline 保留空格读到换行
HFT：cin>> 拆订单号/价/量；getline 读备注与日志
拼接 +/+=  比较 ==  长度 size/empty
vs char*：string 安全；C 接口用 c_str() 过渡
cctype：isalpha/isdigit；参数 unsigned char→int
HFT：校验合约全字母、价格全数字
```

## 示例

```cpp
#include <iostream>
#include <string>
#include <cctype>

int main() {
    std::string s1 = "hello", s2;
    s2 = s1;
    s1 = "world";
    std::cout << s1 << " " << s2 << std::endl;  // world hello

    std::string greeting = s1 + ", " + s2;
    std::cout << greeting << std::endl;

    std::string line;
    std::getline(std::cin, line);

    for (unsigned char c : s1) {
        if (std::isalpha(c))
            std::cout << static_cast<char>(std::toupper(c));
    }
    std::cout << std::endl;

    return 0;
}
```
